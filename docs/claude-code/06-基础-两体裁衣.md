### 4.1 Multi-Agent vs Sub-Agent：更深层次的思考

在第3课中，我们学习了 Sub-Agent 的概念。Multi-Agent 和 Sub-Agent 有什么关系？

**简单理解：**
- **Sub-Agent**：强调将任务分解为多个专用代理，每个代理专注一件事
- **Multi-Agent**：强调多个代理之间的协作、竞争和通信

```
Sub-Agent 是战术层面的概念（如何分解任务）
Multi-Agent 是战略层面的概念（如何组织代理协作）
```

**更准确的区分：**

| 维度 | Sub-Agent | Multi-Agent |
|------|-----------|-------------|
| **核心思想** | 分而治之 | 协作博弈 |
| **代理关系** | 主从关系 | 平等/协作/竞争 |
| **通信方式** | 单向（汇报给主代理） | 多向（互相通信） |
| **典型场景** | 任务分解 | 复杂系统模拟 |
| **设计重点** | 任务分配 | 通信协议 |

### 4.2 Multi-Agent 系统的四大架构模式

#### 模式1：层次结构（Hierarchical）

```
                    Master Agent
                    /    |    \
                   /     |     \
              Sub-A   Sub-B   Sub-C
              /  \       |
            Leaf1 Leaf2  Leaf3
```

**核心特点：**
- Master Agent 负责任务分解和结果汇总
- Sub-Agent 可以进一步分解任务给 Leaf Agent
- 通信是单向的：下级向上级汇报
- 类似于公司的组织架构

**适用场景：**
- 复杂的企业流程（如代码审查 + 发布）
- 需要严格的任务依赖关系
- 结果需要统一汇总

**代码实现：**

```python
class HierarchicalAgent:
    """层次结构 Agent"""

    def __init__(self):
        self.master = MasterAgent()
        self.sub_agents = {
            "security": SecurityAgent(),
            "performance": PerformanceAgent(),
            "style": StyleAgent()
        }
        self.leaf_agents = {
            "sql_checker": SQLInjectionChecker(),
            "xss_checker": XSSChecker(),
            "auth_checker": AuthChecker()
        }

    async def execute(self, task: str) -> Dict:
        """执行任务"""
        # 1. Master 分析任务
        subtasks = await self.master.decompose(task)

        # 2. 分发给 Sub-Agent
        results = {}
        for name, subtask in subtasks.items():
            agent = self.sub_agents[name]
            result = await agent.execute(subtask)
            results[name] = result

        # 3. Master 汇总结果
        return await self.master.aggregate(results)
```

#### 模式2：并行结构（Parallel）

```
              任务分发器
           /    |    \
          /     |     \
    Worker1  Worker2  Worker3
     \|/       \|/       \|/
   结果1     结果2     结果3
          \    |    /
           聚合器
```

**核心特点：**
- 多个 Worker Agent 同时处理独立任务
- 任务分发器负责分解和分发
- 结果聚合器负责汇总
- 类似 MapReduce 的思想

**适用场景：**
- 大量独立任务的批量处理
- 需要并行加速的场景
- 审查多个文件、PR、Issue

**代码实现：**

```python
class ParallelAgent:
    """并行结构 Agent"""

    def __init__(self, max_workers: int = 5):
        self.max_workers = max_workers

    async def execute(self, tasks: List[str]) -> List[Dict]:
        """并行执行多个任务"""
        semaphore = asyncio.Semaphore(self.max_workers)

        async def execute_with_limit(task):
            async with semaphore:
                return await self.process_task(task)

        # 并行执行所有任务
        results = await asyncio.gather(
            *[execute_with_limit(task) for task in tasks],
            return_exceptions=True
        )

        return results

    async def process_task(self, task: str) -> Dict:
        """处理单个任务"""
        # 具体处理逻辑
        pass

    def aggregate(self, results: List[Dict]) -> Dict:
        """聚合结果"""
        valid_results = [r for r in results if not isinstance(r, Exception)]
        return {
            "total": len(results),
            "success": len(valid_results),
            "failed": len(results) - len(valid_results),
            "data": valid_results
        }
```

#### 模式3：管道结构（Pipeline）

```
    Input -> Agent1 -> Agent2 -> Agent3 -> Agent4 -> Output
             解析      转换     验证     生成
```

**核心特点：**
- 严格的顺序依赖，每个 Agent 处理后传递给下一个
- 类似工厂流水线
- 前一个 Agent 的输出是后一个的输入
- 适合有明确步骤的任务

**适用场景：**
- 数据处理流程（ETL）
- 代码转换/迁移
- 文档生成（大纲 -> 初稿 -> 修订 -> 发布）

**代码实现：**

```python
class PipelineAgent:
    """管道结构 Agent"""

    def __init__(self):
        self.stages = [
            ParserAgent(),      # 解析
            TransformerAgent(), # 转换
            ValidatorAgent(),   # 验证
            GeneratorAgent()    # 生成
        ]

    async def execute(self, input_data: Any) -> Any:
        """执行管道"""
        current = input_data
        results = []

        for i, stage in enumerate(self.stages):
            logger.info(f"Pipeline stage {i+1}: {stage.name}")
            current = await stage.process(current)
            results.append(current)

            # 如果某个阶段失败，可以提前终止
            if current.get("error"):
                logger.error(f"Pipeline failed at stage {i+1}")
                break

        return results[-1] if results else None
```

#### 模式4：网状结构（Mesh）

```
        Agent A <-> Agent B
          ^         ^
          |         |
          v         v
        Agent C <-> Agent D
```

**核心特点：**
- Agent 之间可以直接通信
- 没有中央调度器
- 类似 P2P 网络或聊天群
- 每个 Agent 都有自己的目标和行为

**适用场景：**
- 模拟真实世界的协作场景
- 多方协商和博弈
- 分布式问题求解

**代码实现：**

```python
class MeshAgent:
    """网状结构 Agent"""

    def __init__(self, agent_id: str, capabilities: List[str]):
        self.agent_id = agent_id
        self.capabilities = capabilities
        self.peers: Dict[str, 'MeshAgent'] = {}
        self.message_queue = asyncio.Queue()

    def add_peer(self, peer: 'MeshAgent'):
        """添加对等节点"""
        self.peers[peer.agent_id] = peer

    async def send_message(self, to: str, message: Dict):
        """发送消息给对等节点"""
        if to in self.peers:
            await self.peers[to].receive_message({
                "from": self.agent_id,
                **message
            })

    async def broadcast(self, message: Dict):
        """广播消息给所有对等节点"""
        for peer_id in self.peers:
            await self.send_message(peer_id, message)

    async def receive_message(self, message: Dict):
        """接收消息"""
        await self.message_queue.put(message)

    async def process_messages(self):
        """处理消息队列"""
        while True:
            message = await self.message_queue.get()
            await self.handle_message(message)
```

### 4.3 代理拓扑结构设计

#### 如何选择拓扑结构？

```
                    任务特征分析

任务可分解？
├── 是 -> 任务独立？
│   ├── 是 -> 并行结构
│   └── 否 -> 任务有序？
│       ├── 是 -> 管道结构
│       └── 否 -> 层次结构
└── 否 -> 任务复杂？
    ├── 是 -> 需要协作？-> 网状结构
    └── 否 -> 单 Agent 即可
```

#### 拓扑结构的对比

| 拓扑 | 复杂度 | 吞吐量 | 延迟 | 适用场景 |
|------|--------|--------|------|----------|
| 层次结构 | 中 | 中 | 中 | 复杂任务分解 |
| 并行结构 | 低 | 高 | 低 | 批量独立任务 |
| 管道结构 | 中 | 高 | 中 | 有序任务流 |
| 网状结构 | 高 | 可变 | 可变 | 协商博弈 |

#### 通信协议设计

Multi-Agent 系统中，代理间的通信是关键。

```python
# 消息类型定义
class MessageType(Enum):
    REQUEST = "request"           # 请求
    RESPONSE = "response"         # 响应
    BROADCAST = "broadcast"        # 广播
    HEARTBEAT = "heartbeat"        # 心跳
    ERROR = "error"                # 错误

@dataclass
class AgentMessage:
    """Agent 消息"""
    id: str                        # 消息 ID
    type: MessageType              # 消息类型
    from_agent: str                # 发送者
    to_agent: Optional[str]        # 接收者（None 表示广播）
    content: Dict                  # 消息内容
    timestamp: float               # 时间戳
    correlation_id: Optional[str]  # 关联 ID（用于请求-响应匹配）
```

#### 同步 vs 异步通信

**同步通信：**

```python
# 同步请求-响应
async def request(self, target: str, action: str, data: Dict) -> Dict:
    """发送请求并等待响应"""
    message_id = generate_id()

    # 发送请求
    await self.send_message(target, {
        "id": message_id,
        "type": MessageType.REQUEST,
        "action": action,
        "data": data
    })

    # 等待响应
    response = await self.wait_for_response(message_id, timeout=30)
    return response
```

**异步通信：**

```python
# 异步消息队列
class AsyncMailbox:
    """异步邮箱"""

    def __init__(self):
        self.queue = asyncio.Queue()
        self.handlers: Dict[str, Callable] = {}

    def register_handler(self, message_type: str, handler: Callable):
        """注册消息处理器"""
        self.handlers[message_type] = handler

    async def process_messages(self):
        """消息处理循环"""
        while True:
            message = await self.queue.get()
            handler = self.handlers.get(message.type)

            if handler:
                asyncio.create_task(handler(message))
            else:
                logger.warning(f"No handler for message type: {message.type}")
```

### 4.5 代理协作与竞争机制

#### 协作机制

```python
# 协作：共同完成一个目标
class CollaborativeScenario:
    """协作场景"""

    def __init__(self):
        self.agents = {
            "planner": PlannerAgent(),
            "executor": ExecutorAgent(),
            "reviewer": ReviewerAgent()
        }

    async def solve(self, problem: str) -> Dict:
        """协作解决问题"""
        # 1. Planner 分析问题，制定计划
        plan = await self.agents["planner"].create_plan(problem)

        # 2. Executor 执行计划
        result = await self.agents["executor"].execute(plan)

        # 3. Reviewer 检查结果
        review = await self.agents["reviewer"].review(result)

        # 4. 如果有问题，返回给 Planner 重新规划
        if not review["passed"]:
            return await self.solve(review["feedback"])

        return result
```

#### 竞争机制

```python
# 竞争：多个 Agent 争抢任务
class CompetitiveScenario:
    """竞争场景"""

    def __init__(self):
        self.agents = {f"worker_{i}": WorkerAgent() for i in range(5)}
        self.task_queue = asyncio.Queue()

    async def assign_task(self, task: Dict):
        """分配任务给最快的 Agent"""
        # 同时向所有 Agent 发送任务邀请
        responses = await asyncio.gather(
            *[agent.interested(task) for agent in self.agents.values()],
            return_exceptions=True
        )

        # 选择第一个响应的 Agent
        for i, response in enumerate(responses):
            if response and not isinstance(response, Exception):
                agent_id = list(self.agents.keys())[i]
                await self.agents[agent_id].assign(task)
                return

        # 如果没有人响应，任务失败
        raise TaskAssignmentError("No agent available")
```

```python
class MultiAgentSystem:
    """生产级 Multi-Agent 系统"""

    def __init__(self, config: SystemConfig):
        self.config = config

        # 1. 消息代理
        self.broker = MessageBroker(config.broker)

        # 2. Agent 池
        self.agent_pools = {
            "security": AgentPool(SecurityAgent, config.security_pool_size),
            "performance": AgentPool(PerformanceAgent, config.perf_pool_size),
            "style": AgentPool(StyleAgent, config.style_pool_size)
        }

        # 3. 任务编排器
        self.orchestrator = TaskOrchestrator(
            broker=self.broker,
            pools=self.agent_pools
        )

        # 4. 状态存储
        self.storage = StateStore(config.storage)

    async def submit_task(self, task: Task) -> str:
        """提交任务"""
        task_id = generate_id()

        # 1. 分解任务
        subtasks = await self.orchestrator.decompose(task)

        # 2. 存储任务状态
        await self.storage.save_task_state(task_id, {
            "status": "pending",
            "subtasks": subtasks,
            "created_at": time.time()
        })

        # 3. 分发子任务
        for subtask in subtasks:
            await self.orchestrator.dispatch(subtask)

        return task_id

    async def get_task_result(self, task_id: str) -> TaskResult:
        """获取任务结果"""
        state = await self.storage.get_task_state(task_id)

        if state["status"] == "pending":
            # 等待完成
            await self.wait_for_completion(task_id)

        # 汇总结果
        return await self.orchestrator.aggregate(task_id)
```

### 4.7 实战案例：智能代码审查团队

构建一个智能代码审查团队，模拟真实团队的协作模式。

```
审查团队：
├── Team Lead（主代理）
│   ├── Security Expert（安全专家）
│   ├── Performance Expert（性能专家）
│   ├── Style Expert（风格专家）

    2. 分解为：安全、性能、风格、测试覆盖

    4. Team Lead 汇总结果
```

#### 代码实现

```python
class CodeReviewTeam:
    """代码审查团队"""

    def __init__(self):
        self.team_lead = TeamLeadAgent()
        self.experts = {
            "security": SecurityExpert(),
            "performance": PerformanceExpert(),
            "style": StyleExpert(),
            "qa": QAEngineer()
        }

    async def conduct_review(self, code_change: CodeChange) -> ReviewReport:
        """进行代码审查"""
        # 1. Team Lead 分析变更
        review_plan = await self.team_lead.create_plan(code_change)

        # 2. 并行审查
        review_tasks = []
        for expert_name, focus_areas in review_plan.items():
            expert = self.experts[expert_name]
            task = expert.review(code_change, focus_areas)
            review_tasks.append(task)

        results = await asyncio.gather(*review_tasks, return_exceptions=True)

        # 3. Team Lead 汇总
        report = await self.team_lead.compile_report(results)

        # 4. 如果有严重问题，标记需要重新审查
        if report.has_critical_issues:
            report.next_action = "request_changes"
        else:
            report.next_action = "approve"

        return report

# 使用示例
```

### 4.8 Multi-Agent 系统的挑战与解决方案

#### 挑战1：状态一致性

```python
# 问题：多个 Agent 同时修改同一资源
# 解决方案：分布式锁

class DistributedLock:
    """分布式锁"""

    async def acquire(self, resource: str, timeout: int = 30) -> bool:
        """获取锁"""
        lock_key = f"lock:{resource}"
        acquired = await self.redis.set(
            lock_key,
            self.agent_id,
            nx=True,  # 只有不存在时设置
            ex=timeout
        )
        return acquired

    async def release(self, resource: str):
        """释放锁"""
        lock_key = f"lock:{resource}"
        # 只有锁的持有者才能释放
        await self.redis.eval("""
            if redis.call('get', KEYS[1]) == ARGV[1] then
                return redis.call('del', KEYS[1])
            else
                return 0
            end
        """, [lock_key], self.agent_id)
```

#### 挑战2：死锁和活锁

```python
# 死锁：Agent A 等 Agent B，Agent B 等 Agent A
# 解决方案：检测循环依赖，使用超时机制

class DeadlockDetector:
    """死锁检测"""

    def detect_cycle(self, wait_graph: Dict[str, Set[str]]) -> Optional[List[str]]:
        """检测等待图中的环"""
        visited = set()
        path = []

        def dfs(node: str) -> Optional[List[str]]:
            if node in path:
                return path[path.index(node):] + [node]

            if node in visited:
                return None

            visited.add(node)
            path.append(node)

            for neighbor in wait_graph.get(node, []):
                result = dfs(neighbor)
                if result:
                    return result

            path.pop()
            return None

        for node in wait_graph:
            cycle = dfs(node)
            if cycle:
                return cycle

        return None
```

```python
class AgentTracer:

            self.traces[span_id]["events"].append({
                "time": time.time(),
                **event
            })

    async def finish(self, span_id: str, status: str):
        """结束追踪"""
        if span_id in self.traces:
            self.traces[span_id].update({
                "end_time": time.time(),
                "status": status,
                "duration": time.time() - self.traces[span_id]["start_time"]
            })

            # 发送到追踪系统
            await self.export_to_jaeger(self.traces[span_id])
```

### 4.11 总结

> **核心要点回顾：**
> 1. Multi-Agent 强调代理间的协作和通信，是比 Sub-Agent 更高层次的概念
> 2. 四大架构模式：层次结构（主从）、并行结构（MapReduce）、管道结构（流水线）、网状结构（P2P）
> 3. 代理拓扑选择取决于任务特征：是否可分解、是否有依赖、是否需要并行
> 4. 通信机制包括同步请求-响应和异步消息队列
> 5. 生产级 Multi-Agent 系统需要考虑状态一致性、死锁检测、可观测性等挑战
