### 9.1 SKILL.md 是什么？

#### 概念解释

```
SKILL.md = Claude Code 的技能定义文件

类比理解：
- 如果 Claude Code 是一个"AI 助手"
- SKILL.md 就是这个助手的"技能手册"
- 告诉 Claude 在什么情况下使用什么技能
- 以及如何使用这些技能
```

#### Skills vs 普通命令

```
普通命令：
> 帮我写一个 Git 提交信息
-> 每次都需要详细描述需求

Skill：
> /git-commit "添加用户登录功能"
-> 自动按照预定义的规范生成提交信息
```

### 9.2 SKILL.md 标准格式

```markdown
---
name: skill-name                    # 技能名称（唯一标识）
description: "简短描述，说明何时使用"  # 描述
version: "1.0.0"                    # 版本号
author: "Your Name"                 # 作者
tags: ["tag1", "tag2"]              # 标签
trigger: "/command"                 # 触发命令（可选）
---

# Skill Name

## 描述（Description）
详细描述这个技能是什么，以及何时应该使用它。

## 触发条件（Triggers）
列出所有可能触发这个技能的情况。

## 使用方法（Usage）
详细说明如何使用这个技能，包括参数、选项等。

## 示例（Examples）
提供使用示例。

## 注意事项（Notes）
使用时的注意事项和限制。

## 参见（See Also）
相关的其他技能。
```

### 9.3 触发机制详解

```python
class SkillTrigger:
    """技能触发器"""

    def __init__(self):
        self.triggers = []

    def add_trigger(self, trigger: str, skill_id: str):
        """添加触发器"""
        self.triggers.append({
            "trigger": trigger.lower(),
            "skill_id": skill_id
        })

    def match(self, user_input: str) -> Optional[str]:
        """
        匹配用户输入，返回对应的 skill_id

        匹配优先级：
        1. 精确匹配（如 /git-commit）
        2. 前缀匹配（如 git-* 匹配 git-commit）
        3. 模糊匹配（如包含关键词）
        """
        user_input_lower = user_input.lower().strip()

        # 1. 精确匹配
        for t in self.triggers:
            if t["trigger"] == user_input_lower:
                return t["skill_id"]

        # 2. 前缀匹配
        for t in self.triggers:
            if user_input_lower.startswith(t["trigger"]):
                return t["skill_id"]

        # 3. 包含匹配
        for t in self.triggers:
            if t["trigger"] in user_input_lower:
                return t["skill_id"]

        return None


class SkillMatcher:
    """技能匹配器"""

    def __init__(self):
        self.triggers = {
            "exact": [],      # 精确匹配
            "prefix": [],     # 前缀匹配
            "contains": [],   # 包含匹配
            "semantic": []    # 语义匹配
        }

    def register_skill(
        self,
        skill_id: str,
        triggers: List[str],
        match_type: str = "contains"
    ):
        """注册技能触发器"""
        entry = {"skill_id": skill_id, "triggers": triggers}

        if match_type == "exact":
            self.triggers["exact"].append(entry)
        elif match_type == "prefix":
            self.triggers["prefix"].append(entry)
        elif match_type == "contains":
            self.triggers["contains"].append(entry)
        elif match_type == "semantic":
            self.triggers["semantic"].append(entry)

    def match(self, user_input: str) -> List[Dict]:
        """
        匹配用户输入

        Returns:
            匹配到的技能列表，按相关度排序
        """
        results = []
        user_input_lower = user_input.lower()

        # 1. 精确匹配
        for entry in self.triggers["exact"]:
            for trigger in entry["triggers"]:
                if trigger.lower() == user_input_lower:
                    results.append({
                        "skill_id": entry["skill_id"],
                        "match_type": "exact",
                        "confidence": 1.0
                    })

        # 2. 前缀匹配
        for entry in self.triggers["prefix"]:
            for trigger in entry["triggers"]:
                if user_input_lower.startswith(trigger.lower()):
                    results.append({
                        "skill_id": entry["skill_id"],
                        "match_type": "prefix",
                        "confidence": 0.9
                    })

        # 3. 包含匹配
        for entry in self.triggers["contains"]:
            for trigger in entry["triggers"]:
                if trigger.lower() in user_input_lower:
                    results.append({
                        "skill_id": entry["skill_id"],
                        "match_type": "contains",
                        "confidence": 0.7
                    })

        # 按置信度排序
        return sorted(results, key=lambda x: x["confidence"], reverse=True)
```

### 9.4 完整 Skill 示例

#### 示例1：Git Commit Skill

```markdown
---
name: git-commit
description: "自动生成符合 Conventional Commits 规范的提交信息"
version: "1.0.0"
author: "Claude Team"
tags: ["git", "commits", "conventions"]
trigger: "/git-commit"
---

# Git Commit Skill

## 描述

这个技能帮助你生成符合 [Conventional Commits](https://www.conventionalcommits.org/) 规范的 Git 提交信息。

**什么时候使用：**
- 完成了新功能
- 修复了 bug
- 更新了文档
- 重构了代码
- 任何需要提交代码的时候

## 触发条件

- 用户输入包含 `/git-commit`
- 用户输入包含 "提交" + "git"
- 用户请求生成提交信息

## 使用方法

```
/git-commit <type>: <description>

或

/git-commit --type=<type> --description=<description>
```

### 类型（Type）

| 类型 | 描述 | 示例 |
|------|------|------|
| feat | 新功能 | `feat: 添加用户登录功能` |
| fix | 修复 bug | `fix: 修复登录超时问题` |
| docs | 文档更新 | `docs: 更新 README` |
| style | 代码格式 | `style: 格式化代码` |
| refactor | 重构 | `refactor: 提取认证逻辑` |
| test | 测试 | `test: 添加单元测试` |
| chore | 工具/构建 | `chore: 升级依赖` |

## 示例

### 示例1：提交新功能
```
输入：添加了用户注册功能
输出：feat: 添加用户注册功能
```

### 示例2：修复 bug
```
输入：修复了支付失败的问题
输出：fix: 修复支付失败问题
```

### 示例3：多行描述
```
输入：重构了认证模块，使用 JWT 替代 Session
输出：
refactor!: 重构认证模块，使用 JWT 替代 Session

BREAKING CHANGE: 旧版 Session 认证不再支持
```

## 参数说明

- `--type` 或 `--t`：提交类型
- `--description` 或 `--d`：简短描述
- `--body` 或 `--b`：详细描述（可选）
- `--footer` 或 `--f`：脚注信息（可选）
- `--breaking` 或 `--bc`：是否包含 Breaking Change

## 注意事项

1. 描述应该简洁明了，不超过 50 字符
2. 使用祈使语气："添加"而不是"添加了"
3. 不要在描述中包含类型名称
4. 如果修改破坏了向后兼容，使用 `!`（如 `feat!:`）

## 参见

- [Conventional Commits 规范](https://www.conventionalcommits.org/)
- `/git-status` - 查看当前 Git 状态
- `/git-push` - 推送到远程仓库
```

#### 示例2：代码审查 Skill

```markdown
---
name: code-review
description: "执行代码审查并生成审查报告"
version: "1.0.0"
author: "Claude Team"
tags: ["code-review", "quality", "security"]
trigger: "/review"
---

# Code Review Skill

## 描述

这个技能执行全面的代码审查，包括：
- 代码质量分析
- 安全漏洞检测
- 性能问题识别
- 风格规范检查
- 测试覆盖评估

**什么时候使用：**
- PR 提交前
- 代码合并前
- 定期代码审查
- 发现了 bug 需要定位

## 触发条件

- 用户输入包含 `/review`
- 用户输入包含 "审查代码"
- 用户输入包含 "代码审查"

## 使用方法

```
/review [文件或目录]

或

/review --file=<path>
/review --dir=<path>
```

## 审查维度

### 1. 代码质量
- 可读性
- 可维护性
- 复杂度
- 命名规范

### 2. 安全审查
检查以下安全漏洞：
- SQL 注入
- XSS 攻击
- CSRF
- 认证失效
- 敏感数据泄露

### 3. 性能分析
- 数据库查询效率
- 算法复杂度
- 内存使用
- 网络请求优化

### 4. 风格检查
- 代码格式化
- 命名规范
- 注释质量
- 文档完整性

## 示例

### 示例1：审查单个文件
```
/review src/auth/login.py
```

### 示例2：审查整个目录
```
/review src/
```

### 示例3：仅审查安全
```
/review --focus=security src/
```

## 输出格式

```markdown
# 代码审查报告

## 概要
- 文件数：5
- 发现问题：12
- 严重程度：3 高，5 中，4 低

## 问题详情

### 🔴 高 - SQL 注入风险
**文件：** `src/db/queries.py:23`
**问题：**
```python
query = f"SELECT * FROM users WHERE id = {user_id}"
```
**建议：**
```python
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```
```

## 注意事项

1. 审查结果基于静态分析，可能存在误报
2. 建议对每个高严重程度问题进行人工复核
3. 某些警告可能是特定场景下的合理做法

## 参见

- `/security-audit` - 专注安全审查
- `/test` - 运行测试
- `/refactor` - 重构建议
```

### 9.6 Skills 编写最佳实践

#### 1. 清晰简洁的描述

```markdown
# Good
description: "生成符合 Conventional Commits 规范的提交信息"

# Bad
description: "这个技能可以用来生成 git 提交信息，
              它会按照一定的规范来生成，
              适合在需要提交代码的时候使用"
```

#### 2. 准确的触发条件

```markdown
# Good - 明确的触发词
## 触发条件
- `/git-commit`
- "提交代码"
- "生成提交信息"

# Bad - 太模糊
## 触发条件
- "git 相关的"
- "代码相关"
```

#### 3. 丰富的示例

```markdown
# Good - 多种场景的示例
## 示例

### 基本用法
/input: 添加新功能
/output: feat: 添加新功能

### 修复类型
/input: 修复了登录问题
/output: fix: 修复登录问题

### 带详细描述
/input: 重构了认证模块，改为 JWT
/output:
refactor: 重构认证模块为 JWT

BREAKING CHANGE: 旧版 Session 认证不再支持
```

#### 4. 明确的限制说明

```markdown
## 限制
- 不支持多行提交信息
- 不支持 emoji 提交
- 需要 Git 仓库初始化
```
