# Lesson 32 - Plugin 开发完整流程：从创建到 Beta 测试

## 核心概念

Plugin 开发涵盖从项目创建、代码实现、本地测试到 Beta 验证的完整流程。本节总结关键里程碑和质量检查点。

---

## 关键设计

### 开发流程总览

```
项目创建 → 编写代码 → 本地测试 → Beta 验证 → ClawHub 发布
```

### 预提交检查清单

- [ ] package.json 有正确的 `openclaw` 元数据
- [ ] `openclaw.plugin.json` manifest 存在且有效
- [ ] 入口使用 `defineChannelPluginEntry` 或 `definePluginEntry`
- [ ] 所有导入使用聚焦的 `plugin-sdk/<subpath>` 路径
- [ ] 内部导入使用本地模块，不是 SDK self-imports
- [ ] 测试通过：`pnpm test -- <bundled-plugin-root>/my-plugin/`
- [ ] 仓库内 plugin：`pnpm check` 通过

### Beta 版本测试流程

1. **关注 GitHub releases**：订阅 [openclaw/openclaw](https://github.com/openclaw/openclaw/releases)，GitHub `Watch` > `Releases`。Beta tag 格式：`v2026.3.N-beta.1`
2. **立即测试**：Beta tag 出现后尽快测试。稳定版发布前窗口通常只有几小时
3. **报告结果**：在 `plugin-forum` Discord 频道的 plugin thread 发帖，报告 `all good` 或具体问题
4. **问题记录**：标题 `Beta blocker: <plugin-name> - <summary>`，加上 `beta-blocker` label
5. **提交 PR**：标题 `fix(<plugin-id>): beta blocker - <summary>`，关联 issue。Maintainer 会关注这些 PR 并合并
6. **沉默 = 绿灯**：错过窗口则 fix 可能延到下一周期

### Channel Plugin 的特殊考虑

对于非保证在 onboarding/setup 时安装的 channel plugin，应使用：
```typescript
import { createOptionalChannelSetupSurface } from "openclaw/plugin-sdk/channel-setup";
```
这会产生一个 setup adapter + wizard pair，宣告安装需求，在 plugin 安装前对真实配置写入失败关闭。

### Provider Plugin 的 Helper 管理

- Provider-specific helpers 应保留在 package-root seam
- 当 helper 仅在单个 bundled provider 包内有用的，不应提升到 `openclaw/plugin-sdk/*`
- 当前 bundled 示例：Anthropic（Claude stream wrappers）、OpenAI（provider builders）、OpenRouter（provider builder + onboarding）

### Plugin 发布配置（Snippets）

ClawHub 发布的规范 snippets 在 `docs/snippets/plugin-publish/` 中。

## 自我检测

1. **问**：Beta 版本测试的窗口有多长？
   **答**：通常只有几小时，在稳定版发布前。

2. **问**：Beta 测试发现问题时，PR 标题应该是什么格式？
   **答**：`fix(<plugin-id>): beta blocker - <summary>`，如 `fix(github): beta blocker - webhook signature validation`。

3. **问**：如果错过了 Beta 窗口会怎样？
   **答**：修复可能延到下一周期。Maintainer 在 Beta 测试期间关注带有此格式标题的 PR 并合并。

4. **问**：对于可能未安装的 channel plugin，使用什么方法避免配置写入失败？
   **答**：使用 `createOptionalChannelSetupSurface(...)` 来自 `openclaw/plugin-sdk/channel-setup`，会产生 setup adapter + wizard pair。

5. **问**：`pnpm check` 在什么情况下需要通过？
   **答**：仅仓库内 plugin（bundled plugins）需要通过。
