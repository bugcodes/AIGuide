# Lesson 28 - Plugin 概述：能力与类型（Plugin Capabilities Overview）

## 核心概念

**Plugin** 是 OpenClaw 扩展的核心方式。每个 plugin 可以注册任意数量的能力：channel、model provider、tool、hook 等。Plugin 无需加入 OpenClaw 仓库，发布到 ClawHub 后用户通过 `openclaw plugins install clawhub:<package-name>` 安装。

---

## 关键设计

### Plugin 可以注册的能力

| 能力 | 注册方法 | 详细指南 |
|------|----------|----------|
| 文本推理（LLM） | `api.registerProvider(...)` | [Provider Plugins](/plugins/sdk-provider-plugins) |
| CLI 推理后端 | `api.registerCliBackend(...)` | [CLI Backends](/gateway/cli-backends) |
| Channel / 消息 | `api.registerChannel(...)` | [Channel Plugins](/plugins/sdk-channel-plugins) |
| 语音（TTS/STT） | `api.registerSpeechProvider(...)` | Provider Plugins |
| 实时转录 | `api.registerRealtimeTranscriptionProvider(...)` | Provider Plugins |
| 实时语音 | `api.registerRealtimeVoiceProvider(...)` | Provider Plugins |
| 媒体理解 | `api.registerMediaUnderstandingProvider(...)` | Provider Plugins |
| 图片生成 | `api.registerImageGenerationProvider(...)` | Provider Plugins |
| 音乐生成 | `api.registerMusicGenerationProvider(...)` | Provider Plugins |
| 视频生成 | `api.registerVideoGenerationProvider(...)` | Provider Plugins |
| Web fetch | `api.registerWebFetchProvider(...)` | Provider Plugins |
| Web search | `api.registerWebSearchProvider(...)` | Provider Plugins |
| Tool 结果中间件 | `api.registerAgentToolResultMiddleware(...)` | SDK Overview |
| Agent tools | `api.registerTool(...)` | 见 Lesson 30 |
| 自定义命令 | `api.registerCommand(...)` | Entry Points |
| Plugin hooks | `api.on(...)` | [Plugin hooks](/plugins/hooks) |
| 内部事件钩子 | `api.registerHook(...)` | Entry Points |
| HTTP 路由 | `api.registerHttpRoute(...)` | Internals |
| CLI 子命令 | `api.registerCli(...)` | Entry Points |

### 三类主要 Plugin

1. **Channel Plugin**：连接消息平台（Discord、IRC、Telegram 等）
2. **Provider Plugin**：添加模型提供商（LLM、proxy 或自定义端点）
3. **Tool / Hook Plugin**：注册 agent tools、事件 hooks 或服务

### 重要约束

- Plugin 应在 manifest 中声明 `activation.onStartup`，避免意外加载
- 工具名称不能与核心工具冲突（冲突会被跳过）
- 工具注册对象有问题（如缺少 `parameters`）会被跳过并报告到 plugin diagnostics，不会破坏 agent 运行
- 自定义 gateway RPC 方法应使用 plugin-specific 前缀，核心 admin 命名空间（`config.*`、`exec.approvals.*`、`wizard.*`、`update.*`）始终保留

## 自我检测

1. **问**：Plugin 发布到哪里？用户如何安装？
   **答**：发布到 ClawHub，通过 `openclaw plugins install clawhub:<package-name>` 安装。裸包规格（如 `@myorg/openclaw-my-plugin`）在发布切替期间从 npm 安装。

2. **问**：三类主要 Plugin 的区别是什么？
   **答**：Channel Plugin 连接消息平台；Provider Plugin 添加模型提供商；Tool/Hook Plugin 注册 agent tools、事件 hooks 或服务。

3. **问**：工具名称与核心工具冲突会发生什么？
   **答**：冲突的工具会被跳过，不会破坏 agent 运行。

4. **问**：Plugin 的 `activation.onStartup` 为什么重要？
   **答**：应明确声明，避免意外加载。OpenClaw 依赖此配置来决定何时加载 plugin runtime。

5. **问**：自定义 gateway RPC 方法可以使用 `config.*` 前缀吗？
   **答**：不可以。核心 admin 命名空间始终保留，应使用 plugin-specific 前缀。
