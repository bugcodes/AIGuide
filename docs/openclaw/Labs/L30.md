# Lesson 30 - Plugin Hooks 与事件系统

## 核心概念

Plugin Hooks 允许 plugin 在 OpenClaw 的关键生命周期节点插入逻辑，监听和修改行为。Hook 守卫语义决定了对流程的控制程度。

---

## 关键设计

### 主要 Hook 类型

| Hook | 说明 |
|------|------|
| `before_tool_call` | 工具调用前，支持 block/requireApproval |
| `before_install` | plugin 安装前 |
| `message_sending` | 消息发送前，支持 cancel |
| `message_received` | 消息接收后，偏好 typed `threadId` 字段 |
| 自定义 plugin hook | 通过 `api.registerHook(...)` 注册 |

### Hook 守卫语义

**before_tool_call**：
- `{ block: true }` - 终端，阻止低优先级 handlers
- `{ block: false }` - 视为无决策（放行）
- `{ requireApproval: true }` - 暂停 agent 执行，提示用户审批（通过 exec 审批 overlay、Telegram buttons、Discord interactions 或 `/approve` 命令）

**before_install**：
- `{ block: true }` - 终端，阻止低优先级 handlers
- `{ block: false }` - 视为无决策（放行）

**message_sending**：
- `{ cancel: true }` - 终端，阻止低优先级 handlers
- `{ cancel: false }` - 视为无决策（放行）

### /approve 命令

`/approve` 命令同时处理 exec 和 plugin 审批，采用有界回退：
- 当 exec approval id 未找到时，重试同样的 id 通过 plugin approvals
- Plugin approval 转发可通过 `approvals.plugin` 独立配置

### 自定义审批回退检测

如需检测相同的回退情况，优先使用 `isApprovalNotFoundError` from `openclaw/plugin-sdk/error-runtime`，不要手动匹配 approval 过期字符串。

### 消息接收偏好

- 偏好使用 typed `threadId` 字段进行入站 thread/topic 路由
- 保留 `metadata` 用于 channel-specific extras

### 消息发送偏好

- 偏好使用 typed `replyToId` / `threadId` 路由字段
- 避免使用 channel-specific metadata keys

## 自我检测

1. **问**：`before_tool_call` 的 `{ block: true }` 和 `{ block: false }` 行为区别是什么？
   **答**：`{ block: true }` 是终端的，阻止低优先级 handlers；`{ block: false }` 视为无决策，放行。

2. **问**：如何让工具调用暂停并等待用户审批？
   **答**：使用 `{ requireApproval: true }`，会通过 exec 审批 overlay、Telegram buttons、Discord interactions 或 `/approve` 命令提示用户。

3. **问**：`/approve` 命令处理哪两种审批？
   **答**：同时处理 exec 审批和 plugin 审批。采用有界回退：当 exec approval id 未找到时，重试通过 plugin approvals。

4. **问**：如何检测 "approval not found" 错误？
   **答**：使用 `isApprovalNotFoundError` from `openclaw/plugin-sdk/error-runtime`，不要手动匹配字符串。

5. **问**：`message_sending` 的 `{ cancel: true }` 行为是什么？
   **答**：终端的，阻止低优先级 handlers。消息不会发送。
