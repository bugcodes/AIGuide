# Lesson 22 - 子Agent系统

## 核心概念

Subagents 允许在单个会话中派生出独立的 Agent 运行。它们可以处理复杂任务的分解，并在完成后将结果汇报给父 Agent。

## 关键设计

### 1. 派生出处

子Agent 可以在：
- **主会话**中派生出后台任务
- **另一个子Agent**中继续派生（嵌套深度可配置）
- 不同 Agent workspace 中运行

### 2. 生命周期

- **Spawning**：使用 `sessions_spawn` 创建
- **Delivery target**：结果可以返回父会话或直接输出到指定 channel
- **Ended**：父会话收到完成通知

### 3. 上下文传递

子Agent 可以接收：
- 父会话的部分上下文（可配置）
- 自己的 workspace（隔离或共享）
- Skill 配置

### 4. 协调钩子

- `subagent_spawning`：协调子 Agent 路由
- `subagent_delivery_target`：决定结果发往哪里
- `subagent_spawned`：观察子 Agent 启动
- `subagent_ended`：观察子 Agent 完成

## 与其他课时的关联

- **Lesson 12** 多 Agent：子Agent 类似于多 Agent 系统，但运行在父会话上下文中
- **Lesson 15** 插件钩子：`subagent_*` 钩子用于协调子Agent
- **Lesson 14** Model Failover：子Agent 有自己的 model 配置和 fallback
