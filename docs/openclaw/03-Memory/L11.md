# Lesson 11 — Memory 向量记忆系统：Session 会话管理

## 核心概念

OpenClaw 将对话组织为**会话（Session）**，每条消息根据来源（DM、群聊、cron job 等）路由到对应会话。

## 关键设计

### 消息路由

| 来源 | 行为 |
|------|------|
| Direct Messages | 默认共享一个会话 |
| Group Chats | 每个群隔离 |
| Rooms/Channels | 每个房间隔离 |
| Cron Jobs | 每次运行全新会话 |
| Webhooks | 每个 hook 隔离 |

### DM 隔离

**警告**：多用户场景下，默认所有 DM 共享一个会话——Alice 的私信会对 Bob 可见！

修复方案：
```json5
{
  session: {
    dmScope: "per-channel-peer", // 按 channel + sender 隔离
  }
}
```
选项：`main`（默认）、`per-peer`、`per-channel-peer`（推荐）、`per-account-channel-peer`

### 会话生命周期

- **每日重置**（默认）：凌晨 4:00 本地时间开启新会话
- **空闲重置**（可选）：配置 `session.reset.idleMinutes`
- **手动重置**：在聊天中输入 `/new` 或 `/reset`

当每日和空闲重置同时配置时，以先到期的为准。

### 状态存储

- **Store**：`~/.openclaw/agents/<agentId>/sessions/sessions.json`
- **Transcripts**：`~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`

### 会话维护

```json5
{
  session: {
    maintenance: {
      mode: "enforce",  // 启用自动清理
      pruneAfter: "30d",
      maxEntries: 500,
    }
  }
}
```
