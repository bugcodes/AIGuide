# Lesson 07: Hook 系统与生命周期事件

> OpenClaw 学习笔记 | 模块二：Agent Loop

## 核心概念

### Hook 的意义

Hook 是 OpenClaw 的**扩展点**，允许在 agent 执行过程中拦截、修改、添加行为。

两种 Hook 系统：
1. **Internal Hooks（Gateway Hooks）**：事件驱动脚本
2. **Plugin Hooks**：agent/tool 生命周期和 gateway pipeline 的扩展点

## 关键设计

### Internal Hooks

| Hook | 时机 | 用途 |
|------|------|------|
| `agent:bootstrap` | 构建 bootstrap 文件时，系统 prompt 最终确定前 | 添加/移除 bootstrap 上下文文件 |
| `command` hooks | `/new`, `/reset`, `/stop` 等命令事件 | 自定义命令行为 |

### Plugin Hooks（按执行顺序）

**Pre-Model 阶段**：
- `before_model_resolve`：无 `messages`，确定性地覆盖 provider/model
- `before_prompt_build`：有 `messages`，注入 `prependContext`、`systemPrompt` 等

**Pre-Agent 阶段**：
- `before_agent_start`：兼容性 hook

**Turn 处理**：
- `before_agent_reply`：内联 actions 后、LLM 调用前，允许插件声称 turn 并返回合成回复或完全沉默

**Post-Agent 阶段**：
- `agent_end`：检查最终消息列表和 run 元数据

**Tool 生命周期**：
- `before_tool_call`：拦截工具参数
- `after_tool_call`：拦截工具结果

**Compaction**：
- `before_compaction` / `after_compaction`：观察或注解压缩周期

**其他**：
- `before_install`：检查内置扫描结果，阻止 skill/plugin 安装
- `tool_result_persist`：同步转换工具结果后再写入 OpenClaw 拥有的 session transcript
- `message_received` / `message_sending` / `message_sent`：入站/出站消息 hooks
- `session_start` / `session_end`：会话生命周期边界
- `gateway_start` / `gateway_stop`：gateway 生命周期事件

### Hook 决策规则

| Hook | `{block: true}` | `{block: false}` |
|------|----------------|-----------------|
| `before_tool_call` | 终端，停止低优先级处理器 | 无操作，不清除先前的 block |
| `before_install` | 终端，停止低优先级处理器 | 无操作，不清除先前的 block |
| `message_sending` | 终端，停止低优先级处理器 | 无操作，不清除先前的 cancel |

### 特殊生命周期事件

**Agent Loop 可以提前结束的情况**：
- Agent 超时（abort）
- AbortSignal（cancel）
- Gateway 断开或 RPC 超时
- `agent.wait` 超时（仅等待，不停止 agent）

**超时配置**：
- `agent.wait` 默认：30 秒
- Agent 运行时默认：172800 秒（48 小时）
- 模型空闲超时：默认 120 秒（Ollama 等可配置）

### Codex App-Server 兼容性

Codex app-server harness 将 OpenClaw plugin hooks 作为文档化镜像表面的兼容性合约，而 Codex native hooks 仍是独立的较低级别 Codex 机制。

## 输出

通过本次学习，我掌握了 OpenClaw 的 Hook 系统：

1. **双系统并存**：Internal Hooks 用于命令事件，Plugin Hooks 用于生命周期
2. **丰富扩展点**：从模型解析到消息发送的每个阶段都可拦截
3. **优先级与终端**：Block 规则决定多个 hooks 冲突时的行为
4. **超时保护**：多层次超时确保资源不耗尽
5. **兼容性设计**：Codex 集成保留 OpenClaw hooks 契约

Hook 系统是 OpenClaw 可扩展性的核心，允许深度定制 agent 行为而不修改核心代码。
