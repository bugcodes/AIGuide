# Lesson 34 - Plugin 调试与故障排除实战

## 核心概念

Plugin 开发中常见的调试场景包括：runtime 注册失败、工具未被发现、配置错误、CLI 命令不工作等。本节覆盖常见问题的诊断和修复方法。

---

## 关键设计

### 场景 1：Plugin Runtime 注册验证

**问题**：Plugin 已安装但工具不可用。

**诊断步骤**：
```bash
# 检查 runtime 注册状态
openclaw plugins inspect my-plugin --runtime --json

# 检查 plugin 列表
openclaw plugins list
```

**常见原因**：
- `contracts.tools` 中未声明工具
- `activation.onStartup` 未设置为 `true`
- 工具名称与核心工具冲突（被跳过）
- 工具注册对象缺少 `parameters`（被跳过并报告到 diagnostics）

### 场景 2：Manifest 验证

**问题**：Plugin 无法加载。

**检查项**：
1. `openclaw.plugin.json` 是否存在且有效
2. `id` 是否唯一
3. `contracts` 是否声明了所有能力
4. `configSchema` 是否为有效 JSON Schema

### 场景 3：CLI 命令不工作

**问题**：`openclaw demo-plugin ping` 报错。

**验证步骤**：
1. 确认 CLI 已注册：`openclaw plugins inspect my-plugin --runtime --json`
2. 检查 command descriptors 配置
3. 确认 plugin 已启用：`plugins.entries.my-plugin.enabled: true`

### 场景 4：可选工具未激活

**问题**：可选工具无法使用。

**检查项**：
1. 是否在 manifest 中声明 `toolMetadata.<tool>.optional: true`
2. 用户是否在 config 中添加 `tools.allow: ["workflow_tool"]` 或 `tools.allow: ["my-plugin"]`

### 场景 5：导入路径错误

**问题**：Plugin 加载时报导入错误。

**正确做法**：
```typescript
// ✅ 正确
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { createPluginRuntimeStore } from "openclaw/plugin-sdk/runtime-store";

// ❌ 错误（单体 root 已废弃）
import { ... } from "openclaw/plugin-sdk";
```

### 场景 6：Beta 版本兼容性问题

**问题**：升级 OpenClaw 后 Plugin 失效。

**解决步骤**：
1. 检查 `openclaw` 元数据中的 `compat.minGatewayVersion`
2. 对比当前 Gateway 版本
3. 参照 Beta 测试流程报告问题并提交 PR

### 调试命令汇总

| 命令 | 用途 |
|------|------|
| `openclaw plugins list` | 列出所有已安装 plugin |
| `openclaw plugins inspect <id> --runtime --json` | 检查 runtime 注册状态 |
| `openclaw sandbox explain` | 显示沙箱配置和工具策略 |
| `openclaw sandbox list` | 查看所有沙箱运行时 |

## 自我检测

1. **问**：工具注册缺少 `parameters` 会怎样？
   **答**：工具会被跳过并报告到 plugin diagnostics，不会破坏 agent 运行。

2. **问**：如何检查 plugin 的 runtime 注册状态？
   **答**：`openclaw plugins inspect <plugin-id> --runtime --json`。

3. **问**：可选工具需要用户做什么才能使用？
   **答**：需要在 config 中添加 `tools.allow: ["tool-name"]` 或 `tools.allow: ["plugin-id"]`。

4. **问**：导入 `openclaw/plugin-sdk` monolithic root 有什么风险？
   **答**：该路径已废弃，会在将来版本中移除。应使用聚焦的子路径如 `plugin-sdk/plugin-entry`。

5. **问**：Beta 版本发现兼容性问题的标准报告流程是什么？
   **答**：在 Discord `plugin-forum` 频道发帖；开 issue 标题 `Beta blocker: <plugin-name> - <summary>` 并加 `beta-blocker` label；提交 PR 标题 `fix(<plugin-id>): beta blocker - <summary>`。
