# Lesson 26 - 沙箱：工作空间访问与后端配置（Workspace Access & Backend）

## 核心概念

沙箱的工作空间访问控制决定工具能"看见"什么文件。后端配置决定沙箱运行在哪里以及如何与本地工作空间同步。

---

## 关键设计

### 工作空间访问（workspaceAccess）

| 值 | 沙箱看到的内容 |
|---|---|
| `"none"`（默认） | 沙箱私有目录 `~/.openclaw/sandboxes/` |
| `"ro"` | 只读挂载 agent 工作空间到 `/agent`（禁用 write/edit/apply_patch） |
| `"rw"` | 读写挂载 agent 工作空间到 `/workspace` |

> Skill 说明：`read` 工具以沙箱根目录为基础。在 `workspaceAccess: "none"` 时，OpenClaw 会将符合条件的 skill 镜像到沙箱的 `skills/` 目录。在 `"rw"` 时，工作空间中的 skills 可通过 `/workspace/skills` 访问。

### Docker 后端核心配置

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "all",
        backend: "docker",
        scope: "session",
        workspaceAccess: "rw",
        docker: {
          // 额外 bind 挂载（格式：host:container:mode）
          binds: ["/home/user/source:/source:ro"],
          // 网络模式（默认 "none"，无出口）
          network: "none",
          // GPU 支持
          gpus: "all",
          // 一键容器初始化命令（创建后运行一次）
          setupCommand: "apt-get install -y curl jq",
          // 镜像
          image: "openclaw-sandbox:bookworm-slim",
        },
      },
    },
  },
}
```

### OpenShell 后端 - 两种工作空间模式

**mirror 模式（本地权威）**：
- exec 前：同步本地 → 远程 OpenShell
- exec 后：同步远程 → 本地
- 适用场景：本地编辑需要在沙箱中生效；希望沙箱操作后本地工作空间也反映更改

**remote 模式（OpenShell 权威）**：
- 首次创建时从本地 seed 远程工作空间一次
- 之后 exec/read/write/edit/apply_patch 直接作用于远程 OpenShell 工作空间
- OpenClaw **不会**将远程更改同步回本地
- 适用场景：沙箱主要是远程的；希望降低每次 exec 的同步开销；不希望本地编辑意外覆盖远程沙箱状态

### bind 挂载安全

OpenClaw 阻止危险绑定来源：
- 阻止：`docker.sock`、`/etc`、`/proc`、`/sys`、`/dev` 及可能暴露它们的父路径
- 阻止常见凭证目录：`~/.aws`、`~/.cargo`、`~/.config`、`~/.docker`、`~/.gnupg`、`~/.netrc`、`~/.npm`、`~/.ssh`
- 路径验证会规范化后重新解析，即使 symlink 逃脱也会失败关闭
- 敏感挂载（secrets、SSH keys、服务凭证）应使用 `:ro`

### 重要约束

- Docker 沙箱默认**无网络**（`network: "none"`），安装包会失败
- `setupCommand` 需要网络 + 可写根目录 + root 用户
- sandbox exec **不继承**宿主机 `process.env`，需要通过 `docker.env` 或自定义镜像传入 API keys
- `workspaceAccess: "ro"` 时 bind 模式保持独立

## 自我检测

1. **问**：在 Docker 后端中，默认网络配置是什么？这意味着什么？
   **答**：默认 `network: "none"`（无出口）。这意味着容器内无法访问网络，安装包会失败。

2. **问**：`workspaceAccess: "ro"` 时，哪些工具会被禁用？
   **答**：`write`、`edit`、`apply_patch` 会被禁用（只读挂载）。

3. **问**：OpenShell 的 `mirror` 和 `remote` 模式核心区别是什么？
   **答**：`mirror` 模式本地工作空间始终是权威，exec 前后都会同步；`remote` 模式首次 seed 后，远程 OpenShell 工作空间就是权威，不同步回本地。

4. **问**：sandbox exec 是否继承宿主机 `process.env`？如何传入 API keys？
   **答**：不继承。需要通过 `docker.env` 配置或使用包含 API keys 的自定义镜像传入。

5. **问**：bind 挂载中，`~/.ssh` 目录会被阻止吗？
   **答**：会。OpenClaw 阻止常见凭证根目录，包括 `~/.ssh`。
