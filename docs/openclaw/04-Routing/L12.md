# Lesson 12 - 多Agent路由 (Multi-agent Routing)

## 核心概念

**多Agent路由**是 OpenClaw 在一个 Gateway 进程中运行多个**完全隔离**的 Agent 的机制。每个 Agent 有自己的：
- 工作区（workspace）：独立的人格文件（SOUL.md、AGENTS.md、USER.md）
- 状态目录（agentDir）：auth profiles、model registry
- 会话存储（sessions）：聊天历史 + 路由状态

关键理解：**一个 Agent = 一个独立的大脑**，有自己的身份、记忆、会话和认证。

## 关键设计

### 1. 绑定（Binding）机制

Binding 是将**频道账号**路由到特定 Agent 的规则。匹配规则按优先级排序：

1. `peer` — 精确的 DM/群组/频道 ID
2. `parentPeer` — 线程继承
3. `guildId + roles` — Discord 角色路由
4. `guildId` — Discord 服务器
5. `teamId` — Slack
6. `accountId` — 频道账号
7. `accountId: "*"` — 频道级默认
8. **默认 Agent** — fallback 到 `agents.list[].default` 或 `main`

**最具体匹配优先**：如果多个 binding 匹配同一层级，第一个配置顺序的胜出。

### 2. 多账号支持

支持多账号的频道：`whatsapp`, `telegram`, `discord`, `slack`, `signal`, `imessage` 等。

```json5
{
  channels: {
    telegram: {
      accounts: {
        default: { botToken: "..." },
        alerts: { botToken: "..." }
      }
    }
  }
}
```

### 3. 跨 Agent QMD 记忆搜索

通过 `agents.list[].memorySearch.qmd.extraCollections` 让一个 Agent 搜索另一个 Agent 的会话记录。

### 4. Per-agent 沙箱和工具限制

```json5
{
  id: "family",
  sandbox: { mode: "all", scope: "agent" },
  tools: {
    allow: ["read", "exec"],
    deny: ["write", "edit", "browser"]
  }
}
```
