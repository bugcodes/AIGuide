# Lesson 16 — 演练场一：System Prompt 系统提示

## 核心概念

OpenClaw 为每次 Agent 运行构建**自定义系统提示**，归 OpenClaw 所有，不使用默认的 pi-coding-agent 提示。

## 关键设计

### 提示结构（固定章节）

- **Tooling**：结构化工具来源提醒 + 运行时使用指导
- **Execution Bias**：紧凑的跟进指导（当场行动、持续到完成、恢复弱结果、检查可变状态、验证后最终确定）
- **Safety**：简短的安全护栏提醒
- **Skills**（有可用技能时）：告知模型如何按需加载技能指令
- **OpenClaw Self-Update**：用 `config.schema.lookup`、`config.patch`、`config.apply` 检查配置，`update.run` 仅在用户明确请求时运行
- **Workspace**：工作目录
- **Documentation**：本地文档路径
- **Workspace Files（注入）**：引导文件包含在下方
- **Sandbox**（启用时）：沙盒路径和权限说明
- **Current Date & Time**：用户本地时间和时区
- **Reply Tags**：可选的回复标签语法
- **Heartbeats**：心跳提示和确认行为
- **Runtime**：主机、OS、节点、模型
- **Reasoning**：当前可见性级别

### Prompt Modes（提示模式）

运行时为每次运行设置 `promptMode`（非用户配置）：

| Mode | 描述 |
|------|------|
| `full`（默认） | 包含所有章节 |
| `minimal` | 用于子代理，省略 Skills、Memory Recall、OpenClaw Self-Update、User Identity、Reply Tags、Messaging、Silent Replies、Heartbeats |
| `none` | 仅返回基础身份行 |

### Workspace Bootstrap 注入

以下文件被**注入上下文窗口**（每次 turn 都会注入）：
- `AGENTS.md`、`SOUL.md`、`TOOLS.md`、`IDENTITY.md`、`USER.md`、`HEARTBEAT.md`、`BOOTSTRAP.md`（全新工作区）、`MEMORY.md`

> 注意：daily memory 文件（`memory/*.md`）**不属于**正常 bootstrap，在普通 turn 中通过 `memory_search` 和 `memory_get` 按需访问，不计入上下文窗口。

### 提示缓存优化

- 大型稳定内容（Project Context）**在**缓存边界之上
- 易变的 channel/session 部分（Control UI 嵌入、Messaging、Voice、Group Chat Context、Reactions、Heartbeats、Runtime）**在**缓存边界之下

这允许本地后端跨 channel turns 重用稳定的工作区前缀。
