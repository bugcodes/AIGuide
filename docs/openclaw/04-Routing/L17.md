# Lesson 17 — 演练场一：Agent Workspace 工作区配置

## 核心概念

Agent Workspace 是 Agent 的工作目录，存储记忆文件、bootstrap 文件、技能和运行时产物。

## 关键设计

### 默认工作区
```
~/.openclaw/workspace/<agentId>/
```

### 关键文件位置

| 文件 | 路径 | 用途 |
|------|------|------|
| MEMORY.md | 工作区根目录 | 长期记忆 |
| memory/YYYY-MM-DD.md | memory/ 子目录 | 每日笔记 |
| DREAMS.md | 工作区根目录 | 梦境日记 |
| skills/ | skills/ 子目录 | 工作区级技能 |

### 配置文件中的工作区配置

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace"  // 默认工作区根目录
    }
  }
}
```

### 子代理工作区隔离

子代理会话只注入 `AGENTS.md` 和 `TOOLS.md`（其他 bootstrap 文件被过滤），以保持子代理上下文小。

## 与其他课时的关联

- **Lesson 08（memory）** — MEMORY.md 和 daily memory 文件都存在于工作区
- **Lesson 10（dreaming）** — Dream Diary 和 dreaming store 在工作区中
- **Lesson 11（session）** — Session transcript 存储在工作区外（sessions/ 子目录）
- **Lesson 16（system-prompt）** — Bootstrap 注入的工作区文件决定系统提示内容
