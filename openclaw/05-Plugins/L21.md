# Lesson 21 - 代理架构与 ACP Agents

## 核心概念

ACP Agents 是 OpenClaw 运行的外部编码工具链（如 Claude Code、Codex 等）。它们作为独立进程运行，OpenClaw 通过标准输入/输出与它们通信。

## 关键设计

### 1. ACP 协议

ACP（Agent Communication Protocol）是 OpenClaw 与外部 Agent 进程之间的通信协议。通过 stdin/stdout 传递 JSON 消息。

### 2. 会话隔离与路由

会话键格式：`agent:<agentId>:<mainKey>`

每个 Agent 的会话完全隔离，外部 Agent 的操作不会泄露到其他 Agent 的会话中。

### 3. 工具支持

外部 Agent 通常支持：
- 文件系统操作（read/write/exec）
- 浏览器自动化
- 代码搜索和编辑

### 4. 生命周期

1. **Spawn**：OpenClaw 启动外部 Agent 进程
2. **Configure**：传递 workspace、skill 配置、认证信息
3. **Execute**：通过 ACP 协议发送指令，接收结果
4. **Terminate**：任务完成或超时

## 与其他课时的关联

- **Lesson 12** 多 Agent：每个 Agent 可以配置自己的 ACP Agent 集成
- **Lesson 20** Skills：ACP Agent 也有自己的 skill 加载机制
