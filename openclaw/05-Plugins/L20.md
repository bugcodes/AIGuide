# Lesson 20 - Skills 系统

## 核心概念

OpenClaw 使用 **AgentSkills 兼容**的 skill 文件夹来教 Agent 如何使用工具。每个 skill 是一个目录，包含 `SKILL.md` 文件（YAML frontmatter + 指令）。

Skills 位于多层加载路径中，高优先级源覆盖低优先级源。

## 关键设计

### 1. 加载位置和优先级

| 优先级 | 来源 | 路径 |
|-------|------|------|
| 1 | Workspace skills | `<workspace>/skills` |
| 2 | Project agent skills | `<workspace>/.agents/skills` |
| 3 | Personal agent skills | `~/.agents/skills` |
| 4 | Managed/local skills | `~/.openclaw/skills` |
| 5 | Bundled skills | 安装时附带的 |
| 6 | Extra skill folders | `skills.load.extraDirs`（配置） |

同名冲突 → 最高优先级胜出。

### 2. Per-agent vs Shared Skills

多 Agent 场景中：
- **Per-agent**：`workspace/skills` — 仅该 Agent 可见
- **Project-agent**：`workspace/.agents/skills` — 仅该 workspace 的 Agent 可见
- **Personal-agent**：`~/.agents/skills` — 所有 Agent 共享
- **Shared managed/local**：`~/.openclaw/skills` — 所有 Agent 共享

### 3. Agent Skill Allowlists

```json5
{
  agents: {
    defaults: { skills: ["github", "weather"] },
    list: [
      { id: "writer" },              // 继承 github, weather
      { id: "docs", skills: ["docs-search"] },  // 替换默认值
      { id: "locked-down", skills: [] }  // 无 skills
    ]
  }
}
```

非空列表 = 最终集合（不合并默认值）。

### 4. 加载时过滤（Gating）

通过 `metadata.openclaw` 声明依赖：

```markdown
---
name: image-lab
metadata:
  {
    "openclaw": {
      "requires": {
        "bins": ["uv"],
        "env": ["GEMINI_API_KEY"],
        "config": ["browser.enabled"]
      }
    }
  }
---
```

- `requires.bins`：PATH 上必须存在
- `requires.env`：环境变量必须存在或配置提供
- `requires.config`：`openclaw.json` 路径必须为真
- `requires.anyBins`：至少一个必须存在

### 5. SKILL.md 格式

```markdown
---
name: image-lab
description: Generate or edit images...
---
```

必须包含至少 `name` 和 `description`。

可选 frontmatter：
- `user-invocable`：暴露为 slash 命令
- `disable-model-invocation`：不从正常 prompt 加载但仍可作为 slash 命令
- `command-dispatch: "tool"`：slash 命令直接分发到工具

### 6. 安装器规格

```markdown
metadata: {
  "openclaw": {
    "install": [
      { "id": "brew", "kind": "brew", "formula": "gemini-cli" },
      { "id": "npm", "kind": "npm", "pkg": "some-pkg" }
    ]
  }
}
```

偏好顺序：brew > uv > node manager > go > download

### 7. Config Overrides

```json5
{
  skills: {
    entries: {
      "image-lab": {
        enabled: true,
        apiKey: { source: "env", provider: "default", id: "GEMINI_API_KEY" },
        env: { GEMINI_API_KEY: "..." },
        config: { endpoint: "https://..." }
      }
    }
  }
}
```

### 8. Skills Watcher

默认监视 skill 文件夹变化，`SKILL.md` 改变时热重载快照。

## 与其他课时的关联

- **Lesson 12** 多 Agent：每个 Agent 有自己的 skill 路径和 allowlist
- **Lesson 15** 插件：插件可以通过 `openclaw.plugin.json` 附带 skills
- `skills.entries.*.env` 和 `skills.entries.*.apiKey` 注入 host 进程环境（非沙箱）
