# Lesson 23 - 消息路由与 Channel 集成

## 核心概念

消息路由决定入站消息如何到达正确的 Agent。OpenClaw 通过 channel plugins（如 Telegram、Discord、WhatsApp）接收消息，然后根据 binding 规则路由到目标 Agent。

## 关键设计

### 1. 消息流

```
入站消息 → Channel Plugin → 消息解析 → Binding 匹配 → Agent 路由 → 会话恢复/新建 → Agent 处理 → 回复派发
```

### 2. Binding 匹配流程

1. 解析消息元数据（channel、accountId、peer、guildId 等）
2. 按优先级尝试匹配 binding
3. 最具体匹配胜出
4. 无匹配 → 默认 Agent

### 3. Channel 特定配置

每个 channel 有自己的账号和配置：

**Discord**：
- Bot tokens per account
- Guild/Channel allowlists
- Role-based routing

**Telegram**：
- Bot tokens per account
- DM policy（pairing/allowlist）
- Group mention gating

**WhatsApp**：
- 手机号账号
- DM policy
- Group allowlists

### 4. 消息派发

Agent 产生的回复通过 `reply_dispatch` 钩子处理，最终由 channel plugin 发送。

### 5. 会话键

格式：`agent:<agentId>:<mainKey>`

- 直接聊天：按 agent 的 mainKey 折叠
- 群组：保持 thread 上下文

## 与其他课时的关联

- **Lesson 12** 多 Agent：binding 是多 Agent 路由的核心
- **Lesson 15** 插件：`message_received`、`message_sending`、`message_sent` 钩子处理消息流
