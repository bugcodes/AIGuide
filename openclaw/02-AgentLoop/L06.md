# Lesson 06: 上下文窗口与 Compaction

> OpenClaw 学习笔记 | 模块二：Agent Loop

## 核心概念

### 问题

每个模型都有上下文窗口（最大 token 数）。当对话接近限制时，OpenClaw **压缩（Compaction）** 旧消息为摘要，以便继续对话。

### Compaction vs Pruning

| | Compaction | Pruning |
|---|------------|--------|
| **做什么** | 将旧对话摘要 | 修剪旧工具结果 |
| **保存** | 是（存入 session transcript） | 否（仅内存，仅按请求） |
| **范围** | 整个对话 | 仅工具结果 |

## 关键设计

### 工作原理

1. 旧对话 turns 被摘要成紧凑条目
2. 摘要保存在 session transcript
3. 最近消息保持完整

**工具块保持**：
- OpenClaw 保持 assistant 工具调用与其匹配的 `toolResult` 条目配对
- 如果分割点落在工具块内，OpenClaw 移动边界以保持配对

### 自动压缩

- 默认开启
- 在会话接近上下文限制时运行
- 或当模型返回上下文溢出错误时（然后重试）

**溢出检测**：
- `request_too_large`
- `context length exceeded`
- `input exceeds the maximum number of tokens`
- `input token count exceeds the maximum number of input tokens`
- `input is too long for the model`
- `ollama error: context length exceeded`

### 手动压缩

```bash
/compact Focus on the API design decisions
```

### 配置选项

```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6", // 使用不同模型
        keepRecentTokens: 4000,  // 保留最近 token
        notifyUser: true,        // 显示状态消息
        memoryFlush: {
          model: "ollama/qwen3:8b"  // 内存刷新专用模型
        }
      }
    }
  }
}
```

### 后继 Transcript

当 `truncateAfterCompaction: true` 启用时：
- 不重写现有 transcript
- 创建新的活动后继 transcript
- 前一个作为归档 checkpoint 保留

### 内存刷新

压缩前，OpenClaw 可以运行**静默内存刷新** turn，将持久笔记存储到磁盘。

## 输出

通过本次学习，我理解了 OpenClaw 的上下文管理：

1. **摘要而非删除**：保留对话语义，减少 token 使用
2. **自动+手动**：双重保障上下文不溢出
3. **工具块保护**：确保工具调用和结果不分离
4. **可配置性**：多种压缩策略，适应不同场景
5. **内存刷新**：压缩前先保存重要笔记

Compaction 是 OpenClaw 处理长对话的关键机制，确保模型始终有足够的上下文空间。
