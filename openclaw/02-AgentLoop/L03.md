# Lesson 03: Agent Loop 机制

> OpenClaw 学习笔记 | 模块二：Agent Loop

## 核心概念

### 什么是 Agent Loop？

Agent Loop 是 AI Agent 执行任务的完整过程：从接收消息 → 组装上下文 → 模型推理 → 工具执行 → 流式回复 → 持久化。

它是将"一条消息"变成"actions 和最终回复"的权威路径，同时保持会话状态一致。

### 为什么需要序列化？

每个会话（session）有一个**专属车道**，防止：
- 工具/会话竞争
- 会话历史不一致

## 关键设计

### 执行流程

```
RPC 调用 (agent / agent.wait)
       |
       v
验证参数 -> 解析会话 (sessionKey/sessionId) -> 持久化会话元数据
       |
       v
返回 {runId, acceptedAt} (立即响应)
       |
       v
agentCommand 执行:
  1. 解析 model + thinking/verbose/trace defaults
  2. 加载 skills snapshot
  3. 调用 runEmbeddedPiAgent
       |
       v
runEmbeddedPiAgent:
  - 通过 per-session + global queues 序列化运行
  - 解析 model + auth profile
  - 订阅 pi events
  - 流式输出 assistant/tool deltas
  - 超时 -> abort
       |
       v
subscribeEmbeddedPiSession 桥接事件:
  - tool events => stream: "tool"
  - assistant deltas => stream: "assistant"
  - lifecycle events => stream: "lifecycle" (phase: start | end | error)
```

### Entry Points（入口点）

- **Gateway RPC**：`agent` 和 `agent.wait`
- **CLI**：`agent` 命令

### 队列与并发控制

```javascript
// 每个 session key 一个车道
perSessionQueue[sessionKey] = queue

// 可选全局车道（串行化所有会话）
globalLane = queue
```

**Session Write Lock**：
- 保护 transcript 文件写入
- 进程感知 + 文件基础（可跨进程）
- 默认等待 60 秒超时
- 默认不可重入（`allowReentrant: true` 可显式开启）

### Workspace 与 Skills

- Workspace 被解析和创建（沙盒运行可能重定向到沙盒 workspace root）
- Skills 被加载（可从快照重用）
- Skills 注入环境变量和 prompt

### Prompt 组装

System Prompt 构建来源：
1. OpenClaw base prompt
2. Skills prompt
3. Bootstrap context
4. Per-run overrides

模型限制和 compaction 保留 token 被强制执行。

## 输出

通过本次学习，我深入理解了 Agent Loop 的完整执行流程：

1. **快速响应**：RPC 返回 `runId` 后立即响应，agent 异步执行
2. **序列化保障**：per-session + global queues 确保状态一致性
3. **事件驱动**：lifecycle/assistant/tool 三种流事件
4. **超时保护**：Agent 运行时超时（默认 48 小时）和模型空闲超时

Agent Loop 是 OpenClaw 的核心执行引擎，理解它的流程对于调试和优化 agent 行为至关重要。
