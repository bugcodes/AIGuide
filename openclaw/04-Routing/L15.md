# Lesson 15 - 插件钩子 (Plugin Hooks)

## 核心概念

**Plugin hooks** 是 OpenClaw 插件的进程内扩展点，用于拦截 Agent 运行、工具调用、消息流、会话生命周期、子Agent 路由、安装或 Gateway 启动事件。

与 [internal hooks](/automation/hooks)（`/new`、`/reset`、`/stop`、`agent:bootstrap` 等）不同，plugin hooks 是为需要深度定制的插件设计的。

## 关键设计

### 1. 注册方式

```typescript
api.on(
  "before_tool_call",
  async (event) => {
    if (event.toolName !== "web_search") return;
    return { requireApproval: { ... } };
  },
  { priority: 50 }
);
```

- `priority`：越高越先运行
- `timeoutMs`：可选的每钩子预算，超时则中止并继续下一个

### 2. 钩子分类

**Agent Turn**：
- `before_model_resolve` — 覆盖 provider/model
- `agent_turn_prepare` — 消费队列中的插件注入
- `before_prompt_build` — 添加动态上下文
- `before_agent_reply` — **可短路**，返回合成回复或静默
- `before_agent_finalize` — 检查自然最终答案并请求额外一轮
- `agent_end` — 观察最终消息、成功状态、运行时长

**Conversation Observation**：
- `model_call_started/ended` — 观察 sanitized 元数据（无 prompt/response 内容）
- `llm_input` — 观察 provider 输入
- `llm_output` — 观察 provider 输出

**Tools**：
- `before_tool_call` — **重写/阻止/要求批准**
- `after_tool_call` — 观察工具结果
- `tool_result_persist` — **重写**从工具结果产生的 assistant 消息

**Messages**：
- `inbound_claim` — 认领入站消息（合成回复）
- `message_received` — 观察入站内容
- `message_sending` — **重写**或**取消**出站内容
- `message_sent` — 观察出站成功/失败
- `before_dispatch` — 检查/重写出站 dispatch
- `reply_dispatch` — 参与最终 reply-dispatch pipeline

**Sessions**：
- `session_start/end`
- `before/after_compaction`
- `before_reset`

**Subagents**：
- `subagent_spawning/delivery_target/spawned/ended`

**Lifecycle**：
- `gateway_start/stop`
- `cron_changed`
- `before_install`

### 3. Tool Call 策略

`before_tool_call` 可返回：
```typescript
{
  params?: Record<string, unknown>;  // 重写工具参数
  block?: boolean;                    // 阻止执行
  blockReason?: string;
  requireApproval?: {
    title: string;
    description: string;
    severity?: "info" | "warning" | "critical";
    timeoutMs?: number;
    timeoutBehavior?: "allow" | "deny";
    onResolution?: (decision) => Promise<void>;
  };
}
```

### 4. 消息钩子上下文

暴露稳定字段：`ctx.sessionKey`、`ctx.runId`、`ctx.messageId`、`ctx.senderId`、`ctx.trace`、`ctx.traceId`、`ctx.spanId`、`ctx.parentSpanId`、`ctx.callDepth`

### 5. 配置覆盖

```json5
{
  "plugins": {
    "entries": {
      "my-plugin": {
        "hooks": {
          "timeoutMs": 30000,
          "timeouts": {
            "before_prompt_build": 90000,
            "agent_end": 60000
          }
        }
      }
    }
  }
}
```

### 6. Session 扩展和 Next-turn 注入

- `api.registerSessionExtension(...)` 持久化小型 JSON 会话状态
- `api.enqueueNextTurnInjection(...)` 在下一个模型 turn 正好到达一次
