# Lesson 18 — 演练场一：Context 与 Compact 上下文与压缩

## 核心概念

OpenClaw 的上下文管理涉及：系统提示内容裁剪、会话压缩（Compaction）、工具结果修剪（Session Pruning）。

## 关键设计

### Context Engine

`/context list` 和 `/context detail` 命令可检查注入的上下文内容和来源。

### Compaction（压缩）

当对话变长时，OpenClaw 会在压缩前运行一个静默轮次，提醒 Agent 将重要上下文保存到记忆文件（memory flush）。

**记忆 flush 防止上下文丢失**：
- 如果 Agent 在对话中有重要事实还未写入文件，它们会在摘要发生前自动保存

### Session Pruning（会话修剪）

修剪工具结果以控制会话大小，详见 `session-pruning.md`。

### Bootstrap 内容的截断机制

- 单文件最大：`agents.defaults.bootstrapMaxChars`（默认 12000）
- 总 bootstrap 内容：`agents.defaults.bootstrapTotalMaxChars`（默认 60000）
- 截断时显示警告：`agents.defaults.bootstrapPromptTruncationWarning`（off/once/always）
