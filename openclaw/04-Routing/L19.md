# Lesson 19 — 演练场一：Skills 与工具链集成

## 核心概念

Skills 是 Agent 的专用工具指南，通过 `read` 工具按需加载。Skills 让 Agent 掌握特定领域的操作方法，而无需在每次对话中嵌入所有指导。

## 关键设计

### Skills 注入机制

当有符合条件的 skills 时，OpenClaw 注入：
```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

模型使用 `read` 工具加载 `SKILL.md`（工作区、托管或捆绑）。

### Skills 预算控制

- 全局默认：`skills.limits.maxSkillsPromptChars`
- Per-agent 覆盖：`agents.list[].skillsLimits.maxSkillsPromptChars`

### 插件捆绑 Skills

插件捆绑的 Skills 只有在该插件启用时才生效。这让工具插件能暴露更深层的操作指南，而无需将所有指导嵌入每个工具描述。

### 常见的 Skills 类型

| Skill | 用途 |
|-------|------|
| browser-automation | 网页自动化、多步流程、登录检查 |
| github | GitHub issues、PR、CI 日志 |
| session-logs | 搜索和分析自己的会话日志 |
| weather | 获取天气预报 |
| taskflow | 多步骤任务的编排 |
| healthcheck | 主机构安全审计 |

## SKILL.md 编写指南

好的 SKILL.md 应该包含：
- **触发条件**：何时使用这个 skill
- **核心步骤**：清晰的执行流程
- **工具列表**：需要用到的工具（使用 `read` 读取文件时用到的绝对路径）
- **注意事项**：常见错误和边界情况
- **自我检测**：帮助 Agent 验证是否正确理解的测试问题
