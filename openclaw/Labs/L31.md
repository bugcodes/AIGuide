# Lesson 31 - Plugin 快速开始：创建一个 Tool Plugin

## 核心概念

Plugin 通过 `definePluginEntry`（非 channel）或 `defineChannelPluginEntry`（channel）定义入口点。注册 API 通过 `api.registerTool()` 等方法暴露。本节以 Tool Plugin 为例演示完整开发流程。

---

## 关键设计

### 最小 Plugin 结构

**package.json**：
```json
{
  "name": "@myorg/openclaw-my-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./index.ts"],
    "compat": {
      "pluginApi": ">=2026.3.24-beta.2",
      "minGatewayVersion": "2026.3.24-beta.2"
    },
    "build": {
      "openclawVersion": "2026.3.24-beta.2",
      "pluginSdkVersion": "2026.3.24-beta.2"
    }
  }
}
```

**openclaw.plugin.json**：
```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "description": "Adds a custom tool to OpenClaw",
  "contracts": {
    "tools": ["my_tool"]
  },
  "activation": {
    "onStartup": true
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false
  }
}
```

### 入口点代码（index.ts）

```typescript
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { Type } from "@sinclair/typebox";

export default definePluginEntry({
  id: "my-plugin",
  name: "My Plugin",
  description: "Adds a custom tool to OpenClaw",
  register(api) {
    api.registerTool({
      name: "my_tool",
      description: "Do a thing",
      parameters: Type.Object({ input: Type.String() }),
      async execute(_id, params) {
        return { content: [{ type: "text", text: `Got: ${params.input}` }] };
      },
    });
  },
});
```

### 注册可选工具

```typescript
api.registerTool(
  {
    name: "workflow_tool",
    description: "Run a workflow",
    parameters: Type.Object({ pipeline: Type.String() }),
    async execute(_id, params) {
      return { content: [{ type: "text", text: params.pipeline }] };
    },
  },
  { optional: true },  // 用户必须主动添加到 allowlist
);
```

### 导入规范

```typescript
// ✅ 正确：使用聚焦的子路径
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
import { createPluginRuntimeStore } from "openclaw/plugin-sdk/runtime-store";

// ❌ 错误：单体 root（已废弃，会被移除）
import { ... } from "openclaw/plugin-sdk";
```

### 测试与发布

**外部 plugin**：
```bash
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
openclaw plugins install clawhub:@myorg/openclaw-my-plugin
```

**仓库内 plugin**：
```bash
pnpm test -- <bundled-plugin-root>/my-plugin/
```

### CLI 注册

Plugin 可通过 `api.registerCli(...)` 添加根命令：

```typescript
api.registerCli(
  ({ program }) => {
    const demo = program
      .command("demo-plugin")
      .description("Run demo plugin commands");

    demo
      .command("ping")
      .description("Check that the plugin CLI is executable")
      .action(() => {
        console.log("demo-plugin:pong");
      });
  },
  {
    descriptors: [
      {
        name: "demo-plugin",
        description: "Run demo plugin commands",
        hasSubcommands: true,
      },
    ],
  },
);
```

验证：
```bash
openclaw plugins inspect demo-plugin --runtime --json
openclaw demo-plugin ping
```

## 自我检测

1. **问**：非 channel plugin 使用哪个入口点函数？
   **答**：`definePluginEntry`（对于 channel 使用 `defineChannelPluginEntry`）。

2. **问**：可选工具和必需工具在注册时的区别是什么？
   **答**：可选工具使用 `api.registerTool(..., { optional: true })`，需要在 `toolMetadata` 中声明 `optional: true`。

3. **问**：所有导入应该使用什么路径风格？
   **答**：使用聚焦的 `openclaw/plugin-sdk/<subpath>` 路径，禁止使用单体 root。

4. **问**：如何验证 plugin runtime 注册是否成功？
   **答**：使用 `openclaw plugins inspect <plugin-id> --runtime --json`。

5. **问**：Plugin 的工具名称可以与核心工具冲突吗？
   **答**：不可以。冲突的工具会被跳过，不会破坏 agent 运行。
