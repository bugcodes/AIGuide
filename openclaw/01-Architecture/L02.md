# Lesson 02: Wire Protocol 与连接生命周期

> OpenClaw 学习笔记 | 模块一：架构

## 核心概念

### 什么是 Wire Protocol？

Wire Protocol 是 Gateway 与客户端之间的通信规则，定义了：

- 如何建立连接（握手）
- 如何发送请求/响应
- 如何接收事件推送
- 如何进行身份验证

### WebSocket 文本帧

OpenClaw 使用 WebSocket 文本帧传输 JSON 数据，这是比 HTTP 更适合双向通信的选择。

## 关键设计

### 连接生命周期

```
Client                    Gateway
  |                          |
  |-------- req:connect ---->|
  |<------ hello-ok ---------|
  |<---- snapshot -----|----| (presence + health)
  |                          |
  |<---- event:presence -----| (设备上线)
  |<---- event:tick ---------| (心跳)
  |                          |
  |-------- req:agent ------>|
  |<------ ack {runId} ------|
  |<---- event:agent --------| (流式响应)
  |<------ res:agent --------| (最终结果)
```

### 请求/响应格式

```json
// 请求
{"type":"req", "id":"唯一ID", "method":"方法名", "params":{}}

// 响应
{"type":"res", "id":"唯一ID", "ok":true, "payload":{}}
{"type":"res", "id":"唯一ID", "ok":false, "error":{}}
```

### 事件格式

```json
{"type":"event", "event":"事件名", "payload":{}, "seq":1, "stateVersion":1}
```

### 身份验证

两种模式：

1. **Token 模式**：`connect.params.auth.token`
2. **Password 模式**：`connect.params.auth.password`

特殊模式：
- `gateway.auth.allowTailscale: true`：Tailscale Serve 模式，从请求头验证
- `gateway.auth.mode: "none"`：关闭验证（危险！不应用于公共入口）
- `gateway.auth.mode: "trusted-proxy"`：信任代理模式

### 幂等性

对于有副作用的方法（`send`、`agent`），需要**幂等性 key**：

- 防止重复执行（如网络重试导致的重复发送）
- Gateway 维护短期去重缓存

### 设备配对与信任

1. **首次连接**：设备身份 + 配对批准 → Gateway 颁发设备 token
2. **后续连接**：使用设备 token 验证
3. **本地连接**：Loopback 自动批准
4. **签名验证**：所有连接必须签署 `connect.challenge` nonce

签名 payload `v3` 还包含 `platform` 和 `deviceFamily`，Gateway 会固定配对元数据并在元数据变化时要求修复配对。

### Features Discovery

`hello-ok.features.methods` 和 `events` 是发现元数据，不是完整的 API 列表。

## 输出

通过本次学习，我掌握了 OpenClaw 的 Wire Protocol 设计：

1. **握手必须性**：第一帧必须是 `connect`，非 JSON 或非 connect 第一帧会导致硬关闭
2. **三元组模式**：Requests/Responses/Events 三种帧类型
3. **安全机制**：多层次身份验证 + 设备配对 + 签名
4. **幂等性保证**：副作用操作的可重试性

Wire Protocol 是 Gateway 与客户端通信的规则手册，理解它对于调试和扩展 OpenClaw 至关重要。
