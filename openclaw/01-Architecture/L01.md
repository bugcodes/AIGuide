# Lesson 01: Gateway 架构概述

> OpenClaw 学习笔记 | 模块一：架构

## 核心概念

### 什么是 Gateway？

Gateway 是 OpenClaw 的核心守护进程，是整个系统的**唯一消息中枢**。它：

- 拥有所有消息渠道的连接（WhatsApp、Telegram、Slack、Discord、Signal、iMessage、WebChat）
- 通过 WebSocket 提供控制平面 API
- 管理所有会话状态和 transcript 文件
- 是唯一打开 WhatsApp（Baileys）会话的地方

### 为什么需要 Gateway？

在分布式 AI Agent 系统中，多个客户端（macOS App、CLI、Web UI、自动化脚本）需要与同一个 Agent 交互。Gateway 解决了：

1. **统一入口**：所有客户端连接到同一个 Gateway，通过 WebSocket 进行通信
2. **状态管理**：会话、transcript、内存都由 Gateway 统一管理
3. **多渠道整合**：一个地方控制所有消息渠道

## 关键设计

### 架构层次

```
[Clients] --> [Gateway] --> [Providers]
 macOS App    |          - WhatsApp (Baileys)
 CLI          |          - Telegram (grammY)
 Web Admin    |          - Slack
 Automations  |          - Discord
              |          - Signal
              |          - iMessage
              |
         [Canvas Host]
         /__openclaw__/canvas/  (Agent 可编辑的 HTML/CSS/JS)
         /__openclaw__/a2ui/    (A2UI host)
```

### 端口与访问

- **默认端口**：18789
- **默认绑定**：`127.0.0.1:18789`（本地优先）
- **远程访问**：Tailscale 或 SSH 隧道

### 连接类型

| 客户端类型 | 连接方式 | 说明 |
|-----------|---------|------|
| 控制平面客户端 | WebSocket | macOS App、CLI、Web UI |
| Node（设备） | WebSocket + role:node | macOS/iOS/Android/headless |
| WebChat | WebSocket | 静态 UI |

## 输出

通过本次学习，我理解了 OpenClaw 的核心架构设计：

1. **Gateway 作为单一中枢**：所有消息和控制在 Gateway 汇聚
2. **WebSocket 作为通信协议**：统一、高效、支持长连接
3. **分层设计**：控制平面与消息平面分离，Gateway 负责路由和状态
4. **设备配对**：Nodes 通过设备身份和配对机制安全连接

Gateway 是 OpenClaw 架构的起点，后续所有的 Agent Loop、消息路由、流式传输都建立在这个基础之上。
