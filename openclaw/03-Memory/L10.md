# Lesson 10 — Memory 向量记忆系统：Dreaming 梦境系统

## 核心概念

Dreaming 是可选的背景记忆整合系统，通过三个协作阶段将短期信号提升为持久记忆，同时保持过程可解释、可审查。

> ⚠️ Dreaming 默认**禁用**，需要显式启用。

## 关键设计

### 三阶段模型

| 阶段 | 目的 | 持久写入 |
|------|------|---------|
| Light | 排序和暂存近期短期材料 | ❌ |
| Deep | 评分并提升持久候选 | ✅（MEMORY.md）|
| REM | 反思主题和重复想法 | ❌ |

**Light 阶段**：摄入近期每日记忆信号和 recall traces，去重并暂存候选行
**Deep 阶段**：按加权评分排名，达到 minScore、minRecallCount、minUniqueQueries 阈值才提升
**REM 阶段**：构建主题和反思摘要，写入 DREAMS.md

### Deep 排名信号

| 信号 | 权重 | 描述 |
|------|------|------|
| Frequency（频率）| 0.24 | 短期信号累积量 |
| Relevance（相关性）| 0.30 | 平均检索质量 |
| Query diversity（查询多样性）| 0.15 | 触发它的不同查询/日期上下文 |
| Recency（近因）| 0.15 | 时间衰减的新鲜度 |
| Consolidation（巩固度）| 0.10 | 多日复发强度 |
| Conceptual richness（概念丰富度）| 0.06 | snippet/path 的概念标签密度 |

### Dream Diary

人类可读的叙事日记，位于 `DREAMS.md`。每个阶段完成后，memory-core 运行后台子代理轮次并追加简短日记条目。

**重要**：Dream Diary 是给人阅读的，不参与提升。Dreaming 生成的报告 artifacts 被排除在短期提升之外。

###  grounded backfill（基于事实的回填）

从历史 `memory/YYYY-MM-DD.md` 笔记读取，写入 `DREAMS.md`。可用命令：
```bash
openclaw memory rem-backfill --path ./memory --stage-short-term  # 暂存候选
openclaw memory rem-backfill --rollback                         # 回滚
```

### 调度

默认每日凌晨 3:00（`0 3 * * *`）运行完整 sweep（light → REM → deep）。
