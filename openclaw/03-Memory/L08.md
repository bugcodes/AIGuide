# Lesson 08 — Memory 向量记忆系统：Memory 核心架构

## 核心概念

OpenClaw 的记忆系统本质是**基于磁盘的持久化文本存储**，没有任何隐藏状态。模型只"记住"写入磁盘的内容。

系统提供两种记忆文件：
- **MEMORY.md** — 长期记忆，存放持久化事实、偏好和决策
- **memory/YYYY-MM-DD.md** — 每日笔记，存放运行时的上下文和观察
- **DREAMS.md**（可选）— 梦境日记，记录系统整合过程供人工审查

## 关键设计

### 记忆工具
- **`memory_search`** — 语义搜索，即使措辞不同也能找到相关内容
- **`memory_get`** — 读取特定记忆文件或行范围

### 记忆后端
| 后端 | 特点 |
|------|------|
| 内置（默认） | SQLite，支持向量相似度和混合搜索 |
| QMD | 本地优先，支持重排和查询扩展 |
| Honcho | AI 原生，跨会话记忆 |
| LanceDB | 本地 Ollama 嵌入支持 |

### 自动内存 flush
在压缩（compaction）前，OpenClaw 会自动运行一个静默轮次，提醒 Agent 将重要上下文保存到记忆文件。这防止了压缩时的上下文丢失。

### Dreaming 系统
可选的背景记忆整合系统，将短期信号提升为持久记忆。详见 Lesson 10。
