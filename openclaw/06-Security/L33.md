# Lesson 33 - 沙箱安全实战：配置与调试

## 核心概念

本演练场聚焦沙箱配置的实际操作和安全调试。通过常见场景练习掌握沙箱的实战用法。

---

## 关键设计

### 场景 1：最小化启用沙箱

**目标**：为非 main 会话启用基本沙箱隔离。

**配置**：
```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        scope: "session",
        workspaceAccess: "none",
      },
    },
  },
}
```

**验证**：
```bash
openclaw sandbox explain
# 显示有效沙箱模式、工具策略和修复配置键
```

### 场景 2：完整沙箱配置（Docker 后端）

**目标**：为所有会话启用沙箱，带读写工作空间和 GPU 支持。

**配置**：
```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "all",
        backend: "docker",
        scope: "agent",
        workspaceAccess: "rw",
        docker: {
          binds: ["/home/user/shared:/shared:ro"],
          network: "bridge",
          gpus: "all",
          image: "openclaw-sandbox-common:bookworm-slim",
        },
      },
    },
  },
}
```

### 场景 3：OpenShell 远程沙箱（mirror 模式）

**目标**：使用 OpenShell 托管沙箱，保持本地工作空间为权威。

**配置**：
```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "all",
        backend: "openshell",
        scope: "session",
        workspaceAccess: "rw",
      },
    },
  },
  plugins: {
    entries: {
      openshell: {
        enabled: true,
        config: {
          from: "openclaw",
          mode: "mirror",
          remoteWorkspaceDir: "/sandbox",
          remoteAgentWorkspaceDir: "/agent",
        },
      },
    },
  },
}
```

**调试**：
- `openclaw sandbox list` - 查看所有运行时（包括 OpenShell）
- `openclaw sandbox recreate` - 删除并重建沙箱

### 场景 4：调试"工具被阻止"问题

**问题**：工具在沙箱中被阻止，不知道原因。

**诊断步骤**：
1. 运行 `openclaw sandbox explain` 查看有效配置
2. 检查工具策略：`tools` allow/deny 优先于沙箱规则
3. 确认 `tools.elevated` 工具不受沙箱影响
4. 检查 bind 挂载路径是否暴露了敏感目录

**关键原则**：
- 工具策略在沙箱规则之前评估
- `tools.elevated` 是 escape hatch，绕过沙箱使用 `gateway` 或 `node` escape path

### 常见错误与修复

| 错误 | 原因 | 修复 |
|------|------|------|
| `EACCES` 写入 heartbeat | Docker DooD 路径不一致 | 使用宿主机绝对路径 + 相同映射 |
| 安装包失败 | 默认 `network: "none"` | 设置 `docker.network: "bridge"` |
| GPU 不可用 | 未配置 `gpus` | 设置 `docker.gpus: "all"` 或指定 UUID |
| bind 挂载失败 | 暴露了敏感路径 | 检查 `~/.ssh`、`~/.aws` 等被阻止 |

## 自我检测

1. **问**：如何快速查看有效的沙箱配置？
   **答**：`openclaw sandbox explain` 显示有效沙箱模式、工具策略和修复配置键。

2. **问**：Docker DooD 部署中 heartbeat 写入失败如何修复？
   **答**：确保 `openclaw.json` 的 workspace 配置使用宿主机绝对路径，且 Gateway 容器有相同路径的卷映射。

3. **问**：沙箱中工具无法访问网络，最可能的原因是什么？
   **答**：默认 `docker.network: "none"`（无出口），需要设置为 `"bridge"` 或其他允许网络出口的模式。

4. **问**：bind 挂载暴露 `~/.ssh` 会发生什么？
   **答**：OpenClaw 会阻止该 bind 挂载，保护 SSH 凭证不被沙箱访问。

5. **问**：工具策略和沙箱规则的执行优先级是什么？
   **答**：工具 deny/allow 策略在沙箱规则之前评估。如果工具被 deny，沙箱化不会让它复活。
