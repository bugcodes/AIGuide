## 课程完整内容

### 1.1 Claude Code 是什么？

Claude Code 是 Anthropic 官方推出的 **CLI 编程工具**，它不仅仅是一个命令行界面（CLI），而是一个完整的 **AI Agent 编程系统**。

```bash
# 安装 Claude Code
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version

# 启动交互式会话
claude
```

### 1.2 Claude Code vs 传统 IDE

| 维度 | 传统 IDE | Claude Code |
|------|----------|-------------|
| 交互方式 | 点选 + 键盘 | 自然语言对话 |
| 上下文 | 当前文件 | 整个项目 + 对话历史 |
| 自动化 | 固定操作 | 动态规划执行 |
| 学习曲线 | 高（快捷键） | 低（说话就行） |
| 适用场景 | 小改动 | 大型重构/新功能 |

### 1.3 五层架构详解

Claude Code 的架构分为 5 层，每一层都有其独特的功能和设计理念。

```
┌─────────────────────────────────────────────────────────┐
│                    第一层：用户交互层                     │
│         (CLI 命令行 / IDE 插件 / Web 界面)              │
├─────────────────────────────────────────────────────────┤
│                    第二层：任务编排层                     │
│    (Agent Loop / 工具调度 / 决策引擎 / 异常处理)          │
├─────────────────────────────────────────────────────────┤
│                    第三层：上下文管理层                    │
│      (会话上下文 / CLAUDE.md / 长期记忆 / 知识库)          │
├─────────────────────────────────────────────────────────┤
│                    第四层：模型推理层                      │
│        (Claude API / Tool Use / Function Call)          │
├─────────────────────────────────────────────────────────┤
│                    第五层：外部集成层                     │
│           (MCP 协议 / Skills / Plugins)                 │
└─────────────────────────────────────────────────────────┘
```

#### 第一层：用户交互层

用户通过这一层与 Claude Code 进行交互。目前支持三种方式：

1. **CLI 交互模式**（最常用）
```bash
# 启动交互式会话
claude

# 指定任务模式（headless）
claude --task "帮我重构 auth.py 模块"

# 读取项目后对话
claude --project ./my-app
```

2. **IDE 插件模式**（VS Code / JetBrains）
```bash
# VS Code 中通过 Command Palette 调用
# Cmd/Ctrl + Shift + P -> "Claude: Chat"
```

3. **Web 界面模式**（Beta）
```bash
claude --web  # 启动 Web 服务器
```

#### 第二层：任务编排层

这是 Claude Code 的"大脑"，负责：
- 理解用户意图
- 规划执行步骤
- 调度工具执行
- 评估执行结果
- 决定是否继续迭代

#### 第三层：上下文管理层

上下文管理是 Claude Code 区别于简单 ChatBot 的关键：

```python
# 上下文优先级（从高到低）
1. 当前会话消息（session context）
2. CLAUDE.md 配置（project context）
3. 系统提示词（system prompt）
4. 工具返回结果（tool results）
5. 用户历史消息（conversation history）
```

#### 第四层：模型推理层

Claude Code 调用 Claude 模型的能力，包括：
- Tool Use（工具调用）
- Function Calling
- Vision（视觉理解）
- Extended Thinking（深度思考）

```bash
# 指定模型
claude --model opus  # 最强模型，适合复杂任务
claude --model sonnet  # 平衡模式，适合日常任务
claude --model haiku  # 轻量模式，适合简单任务
```

#### 第五层：外部集成层

Claude Code 通过标准协议扩展能力：
- **MCP (Model Context Protocol)**：连接外部工具和数据源
- **Skills**：可复用的技能模块
- **Plugins**：插件系统

### 1.4 Agent Loop：六个核心步骤深度剖析

Agent Loop 是 Claude Code 执行任务的核心机制。理解它，才能真正掌握 Claude Code。

```
┌──────────────────────────────────────────────────────────┐
│                     ① 理解意图                            │
│  用户输入 -> 解析语义 -> 提取关键实体 -> 确定任务类型        │
├──────────────────────────────────────────────────────────┤
│                     ② 规划行动                            │
│  拆解任务 -> 确定步骤 -> 选择工具 -> 评估风险              │
├──────────────────────────────────────────────────────────┤
│                     ③ 调用工具                            │
│  执行 Read/Write/Bash/Grep/Glob -> 捕获输出              │
├──────────────────────────────────────────────────────────┤
│                     ④ 收集结果                            │
│  汇总所有工具输出 -> 格式化结果 -> 准备下一步输入          │
├──────────────────────────────────────────────────────────┤
│                     ⑤ 评估结果                            │
│  判断是否达成目标 -> 检查是否有错误 -> 决定下一步         │
├──────────────────────────────────────────────────────────┤
│                     ⑥ 迭代或结束                         │
│  未达成 -> 返回②继续执行  |  已达成 -> 生成最终回复        │
└──────────────────────────────────────────────────────────┘
```

#### 步骤①：理解意图

Claude Code 会分析用户的输入，提取以下信息：

```python
intent_analysis = {
    "task_type": "refactor | create | debug | explain | search | ...",
    "entities": ["文件路径", "函数名", "变量名", ...],
    "constraints": ["不能修改", "必须使用", "限制条件"],
    "expected_output": "什么样的结果"
}
```

示例：
```
用户输入："帮我重构 src/auth.py 中的登录函数，要求使用 async/await"
```

Claude Code 解析为：
```python
{
    "task_type": "refactor",
    "target_file": "src/auth.py",
    "target_function": "登录函数",
    "requirement": "使用 async/await",
    "scope": "单个文件"
}
```

#### 步骤②：规划行动

基于理解的结果，Claude Code 会生成执行计划：

```python
execution_plan = [
    {"step": 1, "action": "read_file", "target": "src/auth.py", "purpose": "了解当前实现"},
    {"step": 2, "action": "analyze", "target": "login_function", "purpose": "识别同步调用"},
    {"step": 3, "action": "refactor", "target": "login_function", "purpose": "改为 async"},
    {"step": 4, "action": "verify", "target": "changes", "purpose": "确保逻辑一致"}
]
```

#### 步骤③：调用工具

Claude Code 会根据计划调用相应的工具：

| 工具 | 使用场景 | 风险等级 |
|------|----------|----------|
| `Read` | 读取文件内容 | 低 |
| `Glob` | 搜索文件 | 低 |
| `Grep` | 在文件中搜索内容 | 低 |
| `Write` | 创建或覆写文件 | 中 |
| `Edit` | 对文件进行精确修改 | 中 |
| `Bash` | 执行 Shell 命令 | 高 |
| `WebFetch` | 获取网页内容 | 低 |

#### 步骤④：收集结果

工具执行后，Claude Code 会收集所有输出：

```python
tool_results = {
    "read_file": {"success": true, "content": "..."},
    "bash": {"success": true, "output": "..."},
    "grep": {"success": true, "matches": [...]}
}
```

#### 步骤⑤：评估结果

Claude Code 会判断：
1. **是否成功**：工具执行是否有错误
2. **是否完成**：是否达成了用户的目标
3. **是否需要改进**：结果是否符合预期

#### 步骤⑥：迭代或结束

```python
if goal_achieved:
    generate_final_response()
else:
    if should_continue():
        return to_step_2()  # 继续执行
    else:
        explain_difficulty()  # 说明困难
```

### 1.5 Tool Use 机制深度剖析

Tool Use 是 Claude Code 与外部世界交互的桥梁。

#### 内置工具详解

**1. Read 工具**
```python
# 基本用法：读取整个文件
Read(path="src/auth.py")

# Claude Code 实际调用：
# read_file(file_path="src/auth.py")
```

**2. Glob 工具**
```python
# 搜索匹配的文件
Glob(pattern="src/**/*.py")

# 支持的通配符：
# ** - 任意目录
# * - 任意字符
# ? - 单个字符
```

**3. Grep 工具**
```python
# 在文件中搜索内容
Grep(path="src/", pattern="def login")

# 搜索选项：
# - case_sensitive: 大小写敏感
# - regex: 正则表达式
# - context: 显示周围行
```

**4. Write 工具**
```python
# 创建新文件或覆写
Write(path="src/new_file.py", content="#!/usr/bin/env python3\n...")
```

**5. Edit 工具**
```python
# 对文件进行精确修改
Edit(path="src/auth.py", old_string="def login():", new_string="async def login():")
```

**6. Bash 工具**
```python
# 执行 Shell 命令
Bash(command="python -m pytest tests/", timeout=60)
```

#### 安全边界设计

Claude Code 内置多层安全保护：

```yaml
# 第一层：工具白名单
allowed_tools:
  - read_file
  - glob
  - grep
  - write_file
  - edit
  - bash
  - web_fetch

# 第二层：危险操作确认
dangerous_operations:
  - pattern: "rm -rf /"
    action: block
  - pattern: "drop database"
    action: confirm
  - pattern: "git push --force"
    action: warn

# 第三层：权限分级
permission_levels:
  read_only:   # 低风险任务
    - read_file
    - glob
    - grep
  read_write:  # 中等风险任务
    - +write_file
    - +edit
  full_access: # 高风险任务
    - +bash
    - +terminal
```

### 1.6 上下文窗口管理策略

Claude Code 有一个上下文窗口限制（通常是 200K tokens），管理好上下文是高效使用的关键。

#### 上下文污染的常见原因

1. **文件过大**：一次性读取过大的文件
2. **历史过长**：对话历史积累过多
3. **无效信息**：包含太多与任务无关的内容

#### 优化策略

```python
# 策略1：选择性读取
# 不要：Read(path="src/")  # 读取整个目录
# 应该：Glob(pattern="src/**/*.py")  # 先找文件

# 策略2：分批处理
# 对于大文件，分批读取和修改

# 策略3：清理对话历史
# 使用 /clear 或 /compact 命令
```

### 1.8 常见问题解答

**Q1：Claude Code 和 Copilot 有什么区别？**

| 维度 | Copilot | Claude Code |
|------|---------|-------------|
| 交互方式 | 自动补全 | 对话式 |
| 上下文范围 | 当前文件 | 整个项目 |
| 任务类型 | 小改动 | 大型任务 |
| 自主性 | 低（辅助） | 高（代理） |

**Q2：Claude Code 能完全替代程序员吗？**

不能。Claude Code 是一个强大的工具，但它：
- 需要人类设定目标和方向
- 需要人类判断结果的正确性
- 需要人类处理边界情况和创新问题

**Q3：Claude Code 的使用成本是多少？**

使用 Claude Code 需要消耗 Claude API 的 tokens：
- Opus：$15/1M input, $75/1M output
- Sonnet：$3/1M input, $15/1M output
- Haiku：$0.25/1M input, $1.25/1M output
