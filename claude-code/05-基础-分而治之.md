### 3.1 从一个真实场景说起

想象你需要完成以下任务：

> **任务：** "帮我审查 src/ 目录下所有代码的安全漏洞，并生成修复报告"

如果用一个单一的 Agent 来处理，会发生什么？

```
单一 Agent 处理流程：

输入：src/ 目录（假设有 100 个文件，50,000 行代码）

Agent 需要：
1. 读取所有 100 个文件
2. 理解每个文件的业务逻辑
3. 检查 SQL 注入、XSS、CSRF 等多种漏洞
4. 跟踪函数调用关系
5. 生成综合报告

问题：
- 上下文窗口溢出（50,000 行代码 > 上下文限制）
- 注意力分散（需要在多种漏洞模式间切换）
- 执行时间长（需要串行处理所有检查）
- 结果质量不稳定（复杂任务容易遗漏）
```

**解决方案：使用 Sub-Agent**

```
Sub-Agent 处理流程：

任务分解：
├─ Agent 1：安全审计专家 → 只负责 SQL 注入检查
├─ Agent 2：安全审计专家 → 只负责 XSS 检查
├─ Agent 3：代码质量专家 → 只负责认证授权检查
└─ Agent 4：报告生成专家 → 汇总结果，生成报告

每个 Agent：
- 只需要关注一种漏洞类型
- 只需要读取相关代码片段
- 在独立上下文中执行，不会互相干扰
```

### 3.2 Sub-Agent 的核心概念

#### 什么是 Sub-Agent？

**Sub-Agent（子代理）** 是一种将复杂任务分解为多个专用代理的设计模式。每个 Sub-Agent 专注于特定类型的任务，在独立的上下文中执行，然后将结果汇总给主代理（Master Agent）。

```python
# Sub-Agent 的基本结构
sub_agent = {
    "name": "security-auditor",        # 子代理名称
    "role": "安全审计专家",              # 子代理角色
    "expertise": ["sql-injection", "xss", "csrf"],  # 专长领域
    "tools": ["read_file", "grep", "glob"],  # 可用工具
    "constraints": ["禁止修改文件", "只读操作"],  # 约束条件
    "context": "独立上下文，不共享主代理上下文"   # 上下文隔离
}
```

#### 为什么需要 Sub-Agent？

**1. 上下文窗口限制**

Claude Code 的上下文窗口是有限的（通常是 200K tokens）。当任务涉及大量代码时，单个 Agent 无法同时处理所有信息。

```
解决方案：分解任务，让多个 Sub-Agent 并行处理
- 每个 Sub-Agent 只需要关注一部分内容
- 并行执行节省时间
- 结果汇总给主代理
```

**2. 专业化分工**

让专业的人做专业的事。

```
通用 Agent：
- 能做 SQL 注入检查
- 能做 XSS 检查
- 能做性能分析
- 能做代码格式化
-> 都会，但都不精

专用 Sub-Agent：
- 安全 Agent：专注于各种安全漏洞
- 性能 Agent：专注于性能问题
- 风格 Agent：专注于代码规范
-> 各有所长，组合使用
```

**3. 任务依赖关系**

有些任务有严格的依赖关系，需要按顺序执行。

```
任务：迁移数据库 + 验证数据 + 更新代码

方案：
Agent 1（数据库迁移） -> 完成后
Agent 2（数据验证）    -> 验证通过后
Agent 3（更新代码）    -> 完成
```

**4. 并行加速**

多个独立任务可以同时执行。

```
任务：审查 10 个 PR

单 Agent：串行处理，10 个 PR 依次审查
Sub-Agent：并行处理，10 个 Agent 同时审查 10 个 PR
```

### 3.3 什么时候应该使用 Sub-Agent？

Sub-Agent 不是万能的，过度使用会增加系统复杂度。以下是判断标准：

#### 适合使用 Sub-Agent 的场景

| 场景 | 示例 | 原因 |
|------|------|------|
| **任务可分解** | 审查多个文件的安全漏洞 | 每个文件独立审查 |
| **需要不同专业知识** | 代码审查 + 性能分析 | 不同领域需要不同 Agent |
| **上下文过长** | 处理 50,000 行代码 | 分解避免上下文溢出 |
| **并行加速** | 审查 10 个 PR | 同时处理节省时间 |
| **结果交叉验证** | 双重检查敏感操作 | 两个 Agent 独立验证 |

#### 不适合使用 Sub-Agent 的场景

| 场景 | 原因 |
|------|------|
| **简单问答** | 一个问题直接回答即可 |
| **单文件小修改** | 上下文足够，不需要分解 |
| **严格串行依赖** | 步骤必须按顺序执行，分解无意义 |
| **需要共享状态** | 分解后状态不共享，反而复杂 |

#### Sub-Agent 使用决策树

```
任务来了，要用 Sub-Agent 吗？

├── 任务涉及多个独立子任务？
│   ├── 是 -> 考虑使用 Sub-Agent
│   └── 否 -> 单 Agent 足够
│
├── 子任务需要不同专业知识？
│   ├── 是 -> 强烈建议使用 Sub-Agent
│   └── 否 -> 可以考虑按功能分解
│
├── 单一 Agent 会超出上下文限制？
│   ├── 是 -> 必须使用 Sub-Agent
│   └── 否 -> 可以考虑其他方案
│
└── 需要并行执行加速？
    ├── 是 -> 使用 Sub-Agent + 并行
    └── 否 -> 串行 Sub-Agent 或单 Agent
```

### 3.4 Sub-Agent 的设计模式

#### 模式1：功能分解（Functional Decomposition）

按功能将任务分解为多个专用 Agent。

```
任务：代码审查

分解方案：
├─ Agent 1：安全专家 → 安全漏洞检查
├─ Agent 2：性能专家 → 性能问题检查
├─ Agent 3：风格专家 → 代码规范检查
└─ Agent 4：文档专家 → 文档完整性检查

汇总：Agent 5（主代理）→ 汇总所有结果
```

#### 模式2：管道模式（Pipeline）

按执行顺序分解任务，前一个 Agent 的输出是后一个的输入。

```
任务：代码迁移

管道：
Input -> Agent1 -> Agent2 -> Agent3 -> Output
       (分析)   (转换)   (验证)   (报告)

Agent1（分析）：分析原始代码结构
Agent2（转换）：将代码转换为新语言/框架
Agent3（验证）：验证转换后的代码正确性
```

#### 模式3：层次结构（Hierarchical）

主代理管理多个子代理，子代理可以再管理更下一级的代理。

```
                    Master Agent
                   /    |    \
                  /     |     \
            Sub-A    Sub-B    Sub-C
            /  \       |
          Leaf1 Leaf2  Leaf3

Master：任务分解，结果汇总
Sub-A/B/C：管理子任务
Leaf1/2/3：执行具体任务
```

#### 模式4：并行 + 汇总（Parallel + Aggregate）

多个 Agent 并行处理独立任务，结果汇总。

```
任务：审查 10 个文件

并行执行：
Agent1 -> 文件1
Agent2 -> 文件2
...
Agent10 -> 文件10

汇总：Agent汇总 -> 综合报告
```

### 3.5 Sub-Agent 的实现细节

#### Agent 间的通信

Sub-Agent 之间需要通信，常见模式：

**1. 共享存储**

```python
# 使用共享存储传递信息
shared_storage = {
    "task_queue": [...],      # 待处理任务
    "results": {},            # 存储结果
    "status": {}              # 任务状态
}

# Agent 间的交互
def agent1_work():
    result = do_task_1()
    shared_storage["results"]["task1"] = result
    shared_storage["status"]["task1"] = "completed"

def agent2_work():
    # 等待 task1 完成
    while shared_storage["status"].get("task1") != "completed":
        time.sleep(0.1)
    data = shared_storage["results"]["task1"]
    return do_task_2(data)
```

**2. 消息队列**

```python
# 使用消息队列
from queue import Queue

task_queue = Queue()
result_queue = Queue()

def agent1_work():
    # 生产者
    task_queue.put({"type": "security", "file": "src/auth.py"})
    task_queue.put({"type": "security", "file": "src/payment.py"})

def agent2_work():
    # 消费者
    while True:
        task = task_queue.get()
        result = audit_security(task["file"])
        result_queue.put(result)
```

**3. 上下文注入**

```python
# 将子代理的结果注入到主代理的上下文
master_context = {
    "original_task": "审查代码安全",
    "sub_results": []
}

# Agent1 执行
agent1_result = execute_agent("security-auditor", code)
master_context["sub_results"].append(agent1_result)

# Agent2 执行
agent2_result = execute_agent("performance-expert", code)
master_context["sub_results"].append(agent2_result)

# Agent3 汇总
final_report = summarize(master_context["sub_results"])
```

### 3.6 实战案例：构建代码审查系统

#### 需求分析

构建一个自动代码审查系统，包含：
- 安全漏洞检查
- 性能问题检查
- 代码规范检查
- 测试覆盖率检查

#### 系统架构

```
代码审查 Master Agent
│
├── 安全审查 Sub-Agent
│   ├── SQL 注入检查
│   ├── XSS 检查
│   └── 认证授权检查
│
├── 性能审查 Sub-Agent
│   ├── 数据库查询检查
│   ├── 缓存使用检查
│   └── 算法复杂度检查
│
├── 规范审查 Sub-Agent
│   ├── 命名规范检查
│   ├── 注释规范检查
│   └── 代码结构检查
│
└── 测试审查 Sub-Agent
    ├── 单元测试覆盖
    ├── 集成测试覆盖
    └── 测试质量检查
```

#### 核心代码实现

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
from dataclasses import dataclass
from typing import List, Dict, Any
import asyncio

@dataclass
class ReviewResult:
    """审查结果"""
    category: str          # 审查类别
    issues: List[Dict]     # 发现的问题
    severity: str         # 严重程度：critical, high, medium, low
    summary: str          # 总结

class CodeReviewSystem:
    """代码审查系统"""

    def __init__(self):
        self.agents = {
            "security": SecurityReviewAgent(),
            "performance": PerformanceReviewAgent(),
            "style": StyleReviewAgent(),
            "testing": TestingReviewAgent()
        }

    async def review(self, code: str, file_path: str) -> Dict[str, Any]:
        """执行代码审查"""
        tasks = []

        # 创建所有审查任务
        for name, agent in self.agents.items():
            task = asyncio.create_task(
                agent.review(code, file_path)
            )
            tasks.append((name, task))

        # 并行执行所有审查
        results = {}
        for name, task in tasks:
            try:
                result = await task
                results[name] = result
            except Exception as e:
                results[name] = ReviewResult(
                    category=name,
                    issues=[],
                    severity="unknown",
                    summary=f"审查失败: {str(e)}"
                )

        # 汇总结果
        return self.aggregate_results(results)

    def aggregate_results(self, results: Dict[str, ReviewResult]) -> Dict:
        """汇总所有审查结果"""
        all_issues = []
        critical_count = 0
        high_count = 0

        for category, result in results.items():
            for issue in result.issues:
                issue["category"] = category
                all_issues.append(issue)

                if issue["severity"] == "critical":
                    critical_count += 1
                elif issue["severity"] == "high":
                    high_count += 1

        return {
            "total_issues": len(all_issues),
            "critical": critical_count,
            "high": high_count,
            "issues": sorted(all_issues, key=lambda x: self.severity_order(x["severity"])),
            "can_merge": critical_count == 0 and high_count == 0
        }

    def severity_order(self, severity: str) -> int:
        """严重程度排序"""
        order = {"critical": 0, "high": 1, "medium": 2, "low": 3}
        return order.get(severity, 4)
```

#### Sub-Agent 实现示例

```python
class SecurityReviewAgent:
    """安全审查 Agent"""

    def __init__(self):
        self.patterns = {
            "sql_injection": {
                "pattern": r"(execute|query|cursor\.execute).*%s|\".*\$|.*\{.*\}",
                "severity": "critical",
                "message": "可能的 SQL 注入漏洞"
            },
            "hardcoded_secret": {
                "pattern": r"(api_key|password|secret)\s*=\s*[\"'][^\"']+[\"']",
                "severity": "high",
                "message": "硬编码的敏感信息"
            },
            "xss": {
                "pattern": r"innerHTML\s*=|document\.write",
                "severity": "high",
                "message": "可能的 XSS 漏洞"
            }
        }

    async def review(self, code: str, file_path: str) -> ReviewResult:
        """执行安全审查"""
        issues = []

        # 检查各种安全模式
        for name, pattern_info in self.patterns.items():
            matches = self.find_pattern(code, pattern_info["pattern"])
            for match in matches:
                issues.append({
                    "type": name,
                    "severity": pattern_info["severity"],
                    "message": pattern_info["message"],
                    "location": match["line"],
                    "code": match["code"]
                })

        return ReviewResult(
            category="security",
            issues=issues,
            severity=self.calculate_severity(issues),
            summary=f"发现 {len(issues)} 个安全问题"
        )

    def find_pattern(self, code: str, pattern: str) -> List[Dict]:
        """在代码中查找模式"""
        import re
        matches = []
        for i, line in enumerate(code.split("\n"), 1):
            if re.search(pattern, line, re.IGNORECASE):
                matches.append({"line": i, "code": line.strip()})
        return matches

    def calculate_severity(self, issues: List[Dict]) -> str:
        """计算整体严重程度"""
        if any(i["severity"] == "critical" for i in issues):
            return "critical"
        elif any(i["severity"] == "high" for i in issues):
            return "high"
        elif any(i["severity"] == "medium" for i in issues):
            return "medium"
        return "low"
```

### 3.7 Sub-Agent 的最佳实践

#### 最佳实践1：明确的职责划分

每个 Sub-Agent 应该有清晰的职责范围。

```python
# Good：职责明确
security_agent = {
    "name": "security-auditor",
    "role": "安全审计专家",
    "expertise": ["sql-injection", "xss", "csrf"],
    "tools": ["read_file", "grep", "glob"],
    "constraints": ["只读操作", "不修改文件"]
}

# Bad：职责模糊
general_agent = {
    "name": "general",
    "role": "什么都做",
    "expertise": ["everything"],
    "tools": ["all"]
}
```

#### 最佳实践2：独立的上下文

每个 Sub-Agent 应该在独立上下文中执行，避免相互干扰。

```python
# 为每个 Sub-Agent 创建独立上下文
agent1_context = create_context(task="安全审查")
agent2_context = create_context(task="性能审查")

# 并行执行
result1 = execute_agent(security_agent, agent1_context)
result2 = execute_agent(performance_agent, agent2_context)
```

#### 最佳实践3：合理的任务粒度

任务分解要合理，既不能太大（超出上下文），也不能太小（增加通信开销）。

```python
# Good：合理的任务粒度
tasks = [
    "审查 src/features/auth/ 目录（20 个文件）",
    "审查 src/features/order/ 目录（15 个文件）",
    "审查 src/features/payment/ 目录（10 个文件）"
]

# Bad：任务粒度不合理
tasks = [
    "审查 1000 个文件",  # 太大，上下文溢出
    "审查 1 个文件"       # 太小，通信开销大于计算
]
```

#### 最佳实践4：健壮的结果汇总

主代理应该有健壮的结果汇总机制。

```python
def aggregate_results(sub_results: List[ReviewResult]) -> FinalReport:
    """汇总结果，处理各种边界情况"""

    # 1. 过滤无效结果
    valid_results = [r for r in sub_results if r is not None]

    # 2. 处理缺失结果
    if len(valid_results) < len(sub_results):
        missing = len(sub_results) - len(valid_results)
        logger.warning(f"{missing} 个 Agent 执行失败")

    # 3. 统一格式
    formatted = [format_result(r) for r in valid_results]

    # 4. 按严重程度排序
    sorted_issues = sorted(
        formatted,
        key=lambda x: severity_order(x["severity"])
    )

    # 5. 生成最终报告
    return FinalReport(issues=sorted_issues, ...)
```

### 3.8 Sub-Agent 的常见陷阱与解决方案

#### 陷阱1：过度分解

**问题：** 任务分解得过于细致，导致系统复杂度过高。

```python
# 过度分解示例
task = "读取一个文件"
分解为：
- Agent1: 打开文件
- Agent2: 读取第一行
- Agent3: 读取第二行
- Agent4: ...
```

**解决方案：** 遵循"必要的复杂性"原则，只有当单 Agent 确实无法处理时才分解。

#### 陷阱2：上下文污染

**问题：** Sub-Agent 的结果包含大量无关信息，污染主代理上下文。

**解决方案：** Sub-Agent 只返回结构化的结果，不要返回完整日志。

```python
# Bad：返回所有日志
return {
    "logs": ["INFO: 开始检查", "INFO: 检查 SQL 注入", ..., "日志 1000 条"],
    "result": {...}
}

# Good：只返回关键信息
return {
    "issue_count": 5,
    "critical_issues": [...],
    "summary": "发现 5 个问题，其中 2 个严重"
}
```

#### 陷阱3：循环依赖

**问题：** Agent 之间存在循环依赖，导致死锁。

```
Bad：
Agent A 需要 Agent B 的结果
Agent B 需要 Agent C 的结果
Agent C 需要 Agent A 的结果
```

**解决方案：** 确保任务依赖是 DAG（有向无环图）。

```python
# Good：明确的依赖关系
task_graph = {
    "analysis": {"depends_on": []},
    "transform": {"depends_on": ["analysis"]},
    "validate": {"depends_on": ["transform"]},
    "report": {"depends_on": ["analysis", "validate"]}
}
```
