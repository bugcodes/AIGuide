#### 代码组织原则

```python
# 1. 单一职责原则（SRP）
# Good
class UserAuthenticator:
    """只负责用户认证"""
    pass

class UserRepository:
    """只负责用户数据访问"""
    pass

# Bad
class UserManager:  # 职责过多
    def authenticate(self): ...
    def get_profile(self): ...
    def update_settings(self): ...


# 2. 依赖注入
# Good
class Service:
    def __init__(self, repository: UserRepository):
        self.repository = repository  # 依赖通过构造函数注入

# Bad
class Service:
    def __init__(self):
        self.repository = UserRepository()  # 硬编码依赖


# 3. 配置文件管理
# Good
import os

config = {
    "database_url": os.environ.get("DATABASE_URL"),
    "api_key": os.environ.get("API_KEY"),
}

# Bad
config = {
    "database_url": "postgres://user:pass@localhost/db",  # 硬编码
    "api_key": "sk-xxxx",  # 暴露密钥
}
```

### 16.2 Multi-Agent 系统设计原则

#### 设计原则

```
Multi-Agent 系统设计的 SOLID 原则：

S - Single Responsibility（单一职责）
  每个 Agent 只负责一种类型的任务

O - Open/Closed（开闭原则）
  Agent 功能可扩展，但不可修改已有行为

L - Liskov Substitution（里氏替换）
  同类型 Agent 可互相替换

I - Interface Segregation（接口隔离）
  每个 Agent 只暴露必要的接口

D - Dependency Inversion（依赖反转）
  高级策略依赖抽象，而非具体实现
```

#### Agent 设计模式

```python
# 模式1：Strategy Pattern
# 不同策略的 Agent 可互换

class AuditStrategy(ABC):
    @abstractmethod
    def audit(self, code: str) -> AuditResult:
        pass

class SecurityAuditStrategy(AuditStrategy):
    def audit(self, code: str) -> AuditResult:
        # 安全审计逻辑
        pass

class PerformanceAuditStrategy(AuditStrategy):
    def audit(self, code: str) -> AuditResult:
        # 性能审计逻辑
        pass

class AuditAgent:
    def __init__(self, strategy: AuditStrategy):
        self.strategy = strategy

    def execute(self, code: str) -> AuditResult:
        return self.strategy.audit(code)


# 模式2：Decorator Pattern
# 动态增强 Agent 能力

class BaseAgent:
    def execute(self, task: Task) -> Result:
        # 基础执行逻辑
        pass

class LoggingAgent(Decorator):
    """日志增强"""
    def execute(self, task: Task) -> Result:
        logger.info(f"Executing: {task}")
        result = super().execute(task)
        logger.info(f"Completed: {result}")
        return result

class RetryAgent(Decorator):
    """重试增强"""
    def execute(self, task: Task) -> Result:
        for attempt in range(3):
            try:
                return super().execute(task)
            except Exception as e:
                if attempt == 2:
                    raise
        return None
```

### 16.3 规范化开发工作流

#### 标准工作流程

```
开发工作流：

1. 需求分析
   ├─ 理解任务目标
   ├─ 确定验收标准
   └─ 分解任务

2. 编码实现
   ├─ 创建代码框架
   ├─ 实现核心逻辑
   ├─ 添加错误处理
   └─ 编写单元测试

3. 代码审查
   ├─ 安全审查
   ├─ 性能审查
   └─ 风格审查

4. 持续集成
   ├─ 自动化测试
   ├─ 覆盖率检查
   └─ 质量门禁

5. 部署发布
   ├─ 环境配置
   ├─ 部署验证
   └─ 监控告警
```

#### Git 工作流

```bash
# Feature Branch Workflow

# 1. 创建功能分支
git checkout -b feature/user-authentication

# 2. 提交更改
git add .
git commit -m "feat: add user authentication"

# 3. 推送分支
git push origin feature/user-authentication

# 4. 创建 Pull Request
gh pr create --title "feat: add user authentication" \
             --body "实现用户认证功能"

# 5. Code Review 后合并
git checkout main
git merge feature/user-authentication
git push origin main
```
